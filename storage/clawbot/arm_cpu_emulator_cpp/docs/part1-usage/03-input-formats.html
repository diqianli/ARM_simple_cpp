Input Formats

The simulator supports four input formats, each suited for different use cases ranging from full ARM64 binary execution to lightweight text traces.

Format Comparison

Feature ELF Text ChampSim JSON
ISA Coverage ~500 mnemonics ~40 mnemonics N/A (trace format) N/A (trace format)
Capstone Required Yes No No No
Branch Prediction Functional pre-sim None Trace-encoded N/A
Use Case Full simulation Quick debugging Academic research Custom pipelines
CLI Flag -f elf -f text -f champsim -f json

ELF Format Recommended

The ELF format provides the most complete simulation experience. It loads a compiled ARM64 ELF binary and executes it through the full pipeline, decoding instructions via the Capstone v5 disassembler.

Key Features

  • Full ARM64 ISA coverage — approximately 500 mnemonics including ARMv8.0 through ARMv8.7-A extensions
  • SVE and SME support — Scalable Vector Extension and Scalable Matrix Extension instructions
  • Crypto extensions — AES, SHA1, SHA2, and CRC instructions
  • Functional pre-simulation — the FunctionalSim component runs a fast functional simulation before the timing simulation to resolve branch targets, providing accurate branch prediction
  • Symbol resolution — ELF symbol table is parsed for function boundary identification
  • Static and dynamic linking — supports both statically linked and dynamically linked binaries

Usage

# Run an ELF binary
./build/arm_cpu_sim -f elf program.elf

# With custom output and memory size
./build/arm_cpu_sim -f elf program.elf -o result.json -m 512
How it works

The ELF loader parses the binary headers, loads code and data sections into simulated memory, then Capstone v5 disassembles each instruction. A functional pre-simulation pass resolves all branch targets before the timing simulation begins, ensuring accurate pipeline behavior.

Text Format

The text format is a simple, human-readable instruction trace format. It is ideal for quick debugging, minimal testing, and cases where you do not want to depend on Capstone or a cross-compiler.

ISA Coverage

The text parser supports approximately 40 mnemonics covering the most common instruction categories:

  • Integer ALU: ADD, SUB, MUL, AND, ORR, EOR, LSL, LSR, ASR, MOV, MVN, NEG
  • Load/Store: LDR, STR, LDP, STP, LDUR, STUR
  • Branch: B, BL, CBZ, CBNZ, B.EQ, B.NE, B.LT, B.GT, B.LE, B.GE, RET
  • Basic FP: FADD, FSUB, FMUL, FDIV

Line Format

OPCODE [dest,] src1[, src2]  [memory_addr]  [branch_target]

Example Text Trace

ADD X0, X1, X2
SUB X3, X4, X5
MUL X6, X7, X8
AND X9, X10, X11
ORR X12, X13, X14
EOR X15, X16, X17
LSL X18, X19, #3
LSR X20, X21, #5
MOV X22, X23
LDR X0, [X1, #0]
LDR W0, [X1, #4]
STR X2, [X3, #8]
LDP X4, X5, [X6, #16]
STP X7, X8, [X9, #24]
B label

CBZ X0, label

FSUB S0, S1, S2
FMUL D3, D4, D5
RET

Usage

./build/arm_cpu_sim -f text trace.txt -o result.json
No branch prediction in text mode

The text format does not include branch target information, so branch prediction is not available. All branches are resolved as-taken in the timing simulation. For accurate branch behavior, use the ELF format.

ChampSim Format

The ChampSim format is an academic trace format widely used in computer architecture research for cache and branch predictor evaluation. It encodes memory access patterns and branch outcomes as compact binary records.

Variants

  • champsim — Raw (uncompressed) ChampSim trace files
  • champsim_xz — XZ-compressed ChampSim trace files (requires liblzma at build time)

Usage

# Raw ChampSim trace
./build/arm_cpu_sim -f champsim trace.champsim

# XZ-compressed ChampSim trace
./build/arm_cpu_sim -f champsim_xz trace.champsim.xz
XZ compression dependency

The champsim_xz format requires the liblzma development library to be installed on the build system. On Ubuntu/Debian: apt install liblzma-dev. On macOS: brew install xz.

JSON Format

The JSON format accepts a pre-formatted instruction trace in JSON structure. This is useful for generating traces programmatically or converting from other simulation tools.

Usage

./build/arm_cpu_sim -f json trace.json -o result.json

Choosing the Right Format

Scenario Recommended Format Reason
Full program simulation with accurate pipeline behavior elf Complete ISA coverage, functional pre-simulation for branch targets
Quick debugging or testing a specific instruction sequence text No cross-compiler or Capstone dependency needed, easy to hand-write
Cache hierarchy or branch predictor research champsim Standard academic format, widely available trace corpora
Custom trace generation from another tool json Structured format, easy to generate programmatically
