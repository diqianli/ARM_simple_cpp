Benchmarks

A suite of benchmark programs for evaluating different aspects of the simulator's performance and microarchitectural behavior.

The project includes a suite of benchmark programs in the benchmarks/ directory. Each benchmark has a C source file (_source.c or .c), a compiled ARM64 binary (_aarch64), and a GEM5 SE variant (_gem5_se).

Benchmark List

1. Dhrystone

benchmarks/dhrystone/

Classic integer benchmark for general-purpose performance.

  • Tests: Integer arithmetic, control flow, string handling
  • Good for: Baseline IPC measurement
./build/arm_cpu_sim -f elf benchmarks/dhrystone/dhrystone_aarch64

2. Fibonacci

benchmarks/fibonacci/

Recursive Fibonacci computation.

  • Tests: Branch prediction, data dependencies, call/return overhead
  • Good for: Evaluating branch predictor accuracy
./build/arm_cpu_sim -f elf benchmarks/fibonacci/fibonacci_aarch64

3. MatMul 4x4

benchmarks/matmul/

4x4 matrix multiplication.

  • Tests: Multiply-accumulate, memory access patterns
  • Good for: FP/SIMD and memory subsystem evaluation
./build/arm_cpu_sim -f elf benchmarks/matmul/matmul_aarch64

4. QuickSort

benchmarks/qsort/

QuickSort on an array of integers.

  • Tests: Branch prediction (partition), recursive calls
  • Good for: Branch prediction challenge, data-dependent behavior
./build/arm_cpu_sim -f elf benchmarks/qsort/qsort_aarch64

5. LinkedList

benchmarks/linkedlist/

Traversal and manipulation of a linked list.

  • Tests: Irregular memory access patterns
  • Good for: Cache performance evaluation, pointer chasing
./build/arm_cpu_sim -f elf benchmarks/linkedlist/linkedlist_aarch64

6. MemcpyBench

benchmarks/memcpy_bench/

Memory copy benchmark.

  • Tests: Pure memory bandwidth and throughput
  • Good for: Memory subsystem stress testing
./build/arm_cpu_sim -f elf benchmarks/memcpy_bench/memcpy_bench_aarch64

Running All Benchmarks

You can iterate over all benchmarks in a loop:

for bench in benchmarks/*/*.c; do
    echo "=== $bench ==="
    # Compile and run each benchmark
done

Comparing Results

Use different presets to compare performance across configurations:

  • Compare presets: --preset default vs --preset high_performance
  • Compare cache configurations: --l1-size 32 vs --l1-size 128
  • Track IPC, cache hit rates, and branch prediction accuracy across benchmarks
Analyzing cross-benchmark trends

Run each benchmark with the same configuration and collect the output JSON files. You can then compare IPC, cache miss rates, and branch misprediction rates to understand how different microarchitectural features affect each workload.
