Automation Scripts

Build, compile, test, and compare using the project's suite of automation scripts designed for development and CI workflows.

1. scripts/build.sh — Build Script

One-click build that handles CMake configuration and compilation.

  • Usage: ./scripts/build.sh [BuildType]
  • BuildType: Debug, Release (default), RelWithDebInfo, MinSizeRel
  • Auto-detects parallel jobs via nproc / sysctl -n hw.ncpu
  • Output binary: build/arm_cpu_sim
./scripts/build.sh           # Release build
./scripts/build.sh Debug     # Debug build with symbols

2. scripts/compile_test_elf.sh — Cross-Compile Test Program

Compiles scripts/test_elf_source.c into tests/data/test_elf_aarch64.

  • Auto-detects cross-compiler from: aarch64-elf-gcc, aarch64-linux-gnu-gcc, aarch64-none-elf-gcc
  • Usage: ./scripts/compile_test_elf.sh [compiler]
  • Compilation flags: -static -O2 -nostdlib
  • Also compiles dynamic variant if using linux-gnu compiler
./scripts/compile_test_elf.sh                    # Auto-detect
./scripts/compile_test_elf.sh aarch64-elf-gcc    # Specify compiler

3. scripts/test_all.sh — Full Test Suite

Comprehensive 6-stage test pipeline for validating the entire simulator.

  • Usage: ./scripts/test_all.sh [--fast]
  • --fast skips stress tests

Test Stages

Stage Description
1. BUILD CMake configure + build + binary check
2. UNIT TESTS ctest with output-on-failure
3. TEXT TRACE SIMULATION Basic, memory, and branch traces
4. ELF SIMULATION Static and dynamic ELF
5. MULTI-CONFIG Minimal, high-performance, tiny L1, large cache configurations
6. STRESS Long trace (1M cycles), memory trace (1000 instructions, 500K cycles)

Each stage is independent — a failure in one stage does not stop subsequent stages. Results are reported as PASS/FAIL/SKIP with a summary at the end.

./scripts/test_all.sh       # Full test suite
./scripts/test_all.sh --fast # Skip stress tests

4. scripts/compare_gem5.py — GEM5 Comparison

Compares simulator output with GEM5 O3 ARM simulator results.

  • Use with --preset gem5_o3_arm_v7a for aligned configuration
  • Comparison dimensions: cycle count, instruction count, IPC, execution time
python3 scripts/compare_gem5.py --ours output.json --gem5 gem5_output.json

CI Integration

All scripts are self-contained and suitable for CI pipelines:

  • test_all.sh returns exit code 0 on success, 1 on any failure
  • Combine with build.sh for a complete CI workflow
Example CI workflow
# In your CI pipeline
./scripts/build.sh Release
./scripts/test_all.sh --fast
echo "CI exit code: $?"
