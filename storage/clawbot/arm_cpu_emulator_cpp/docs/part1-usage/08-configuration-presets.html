Configuration Presets

Built-in CPU microarchitecture presets for common simulation scenarios, from balanced general-purpose to GEM5 cross-validation.

Overview

CPUConfig provides several presets for common use cases. Select a preset via the CLI:

./build/arm_cpu_sim --preset <name> -f elf program.elf

Available presets: default, high_performance, minimal, gem5_o3_arm_v7a.

Mixing presets with overrides

After selecting a preset, any parameter can be overridden individually via CLI flags. For example: --preset gem5_o3_arm_v7a --window-size 64 starts from the GEM5 preset but changes the ROB size to 64.

gem5_o3_arm_v7a()

Matches GEM5's O3_ARM_v7a default configuration for cross-validation. This preset enables direct comparison of simulation results between our simulator and GEM5.

No L3 in GEM5 O3_ARM_v7a

GEM5's O3_ARM_v7a configuration does not include an L3 cache. Our simulator keeps the default L3 (8 MB, 16-way) when using this preset. To disable L3 for a closer match, override with --l3-size 0.

DDR Memory Controller

DDR memory controller settings are shared across all presets unless explicitly overridden:

Setting Value Description
Base latency150 cyclesDefault access latency for a memory request
Row buffer hit bonus-30 cyclesLatency reduction when the requested row is already open
Bank conflict penalty+20 cyclesAdditional latency when two requests target the same bank
Number of banks8Number of independent DRAM banks
Overriding DDR settings

DDR parameters can be overridden via CLI flags just like other configuration options. For example, to model a faster memory subsystem: --ddr-base-latency 100 --num-banks 16.
