Visualization

Explore the Konata pipeline timeline viewer to inspect instruction-level timing, pipeline stalls, and functional unit utilization.

Konata Pipeline Viewer

The simulator includes a Konata-style pipeline timeline viewer that renders per-instruction pipeline stage timing as an interactive, zoomable chart.

Opening the Viewer

There are two ways to access the visualization:

Option A: Open the HTML file directly

file:///Users/mac/storage/clawbot/arm_cpu_emulator_cpp/tools/viz_server/index.html

Option B: Use the built-in web server

./build/arm_cpu_sim -f elf program.elf -p 3000

Then open http://localhost:3000 in your browser. The visualization page loads automatically with the output data served at the same address.

Loading Data

Use the file picker in the visualization interface to load an output.json file generated by the simulator.

Timeline Features

The pipeline timeline supports the following interactions:

Action How
Zoom in/out Mouse wheel or zoom buttons
Pan left/right Click and drag on the timeline
Search instructions Search by PC address to locate specific instructions

Color Coding by Instruction Type

Instructions are color-coded by their type for quick visual identification:

Color Category Examples
Blue ALU operations ADD, SUB, MUL, AND, ORR, EOR
Green Load operations LDR, LDP, LDUR, LDRB, LDRH
Orange Store operations STR, STP, STUR, STRB, STRH
Red / Pink Branch operations B, BL, CBZ, CBNZ, TBZ, RET, BR
Purple Floating Point / SIMD FADD, FSUB, FMUL, FADD (vector), SIMD ops

Pipeline Stages

The timeline shows each instruction's progress through the pipeline stages:

IF (Fetch) → DE (Decode) → RN (Rename) → DI (Dispatch) → IS (Issue) → EX (Execute) → ME (Memory) → WB (Writeback) → RR (Retire)
  • IF (Fetch) — Instruction fetched from the instruction stream
  • DE (Decode) — Instruction decoded and classified
  • RN (Rename) — Register renaming to eliminate false dependencies
  • DI (Dispatch) — Dispatched to the issue queue
  • IS (Issue) — Issued to a functional unit when operands are ready
  • EX (Execute) — Execution on the assigned functional unit
  • ME (Memory) — Memory access (load/store instructions only)
  • WB (Writeback) — Result written back to the register file
  • RR (Retire) — Instruction commits in program order

Performance Metrics Panel

The visualization page includes a metrics panel that displays:

  • IPC — Overall instructions per cycle
  • Cache hit rates — L1, L2, L3 cache hit rates
  • Branch prediction accuracy — Percentage of correctly predicted branches
  • FU utilization — Breakdown of functional unit busy time by type

Alternative: Performance Stats View

For a tabular metrics display, the perf_stats_view.js component provides a structured table view of performance statistics. This is useful when you need to compare numerical values directly rather than viewing the timeline.

Kanata Log Export

The simulator can export the execution trace in Kanata format for use with other visualization tools that support the Kanata log format. This is useful for cross-tool analysis and sharing results with collaborators who use different tooling.
