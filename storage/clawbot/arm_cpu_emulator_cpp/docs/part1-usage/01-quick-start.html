Quick Start

Get the ARM CPU Emulator built, compiled, and running your first simulation in under five minutes.

Prerequisites

The simulator requires the following tools to build and run:

Tool Minimum Version Purpose
CMake 3.20+ Build system configuration
C++ Compiler Clang 14+ / GCC 12+ C++20 language support
aarch64-elf-gcc Any recent version Cross-compiling test ARM64 programs (optional for text trace input)
Dependencies are fetched automatically

All library dependencies (nlohmann/json, spdlog, GoogleTest, ankerl::unordered_dense, Capstone v5) are downloaded via CMake FetchContent. You do not need to install them manually.

Build the Simulator

Option A: One-Click Build (Recommended)

The project provides a convenience script that handles CMake configuration and compilation with auto-detected parallelism:

# Release build (default)
./scripts/build.sh

# Debug build (with symbols, no optimizations)
./scripts/build.sh Debug

Option B: Manual CMake Build

If you prefer to run CMake directly:

# Configure
cmake -B build -DCMAKE_BUILD_TYPE=Release

# Build (uses all available cores)
cmake --build build -j$(nproc)
Build output

The compiled binary is placed at build/arm_cpu_sim. Test binaries (test_config, test_ooo, test_simulation) are also in the build/ directory.

Compile a Test Program

The simulator executes ARM64 ELF binaries. Use the provided script to compile the built-in test source:

./scripts/compile_test_elf.sh

This script auto-detects the cross-compiler by searching for aarch64-elf-gcc, aarch64-linux-gnu-gcc, or aarch64-none-elf-gcc in your PATH. The compiled binary is placed at tests/data/test_elf_aarch64.

Cross-compiler installation (macOS)

On macOS, install aarch64-elf-gcc via Homebrew: brew install aarch64-elf-gcc

Run Your First Simulation

Execute the simulator with the compiled test ELF binary:

# Input file: tests/data/test_elf_aarch64
# Output file: output.json (in current directory)
./build/arm_cpu_sim -f elf tests/data/test_elf_aarch64

Upon completion, the simulator writes results to output.json in the current working directory. This file contains simulation statistics (IPC, CPI, cache hit rates, instruction mix, cycle-by-cycle pipeline trace, and Konata visualization data).

Always note your file paths

When reporting simulation results, always specify both the input file path and the output file path so results are reproducible.

Customize the Output

Use -o to specify a custom output path:

./build/arm_cpu_sim -f elf tests/data/test_elf_aarch64 -o results/my_sim.json

Use -n to limit the number of instructions simulated (useful for quick testing):

./build/arm_cpu_sim -f elf tests/data/test_elf_aarch64 -n 1000

Open the Visualization

The simulator generates Konata-compatible pipeline timeline data. View it in two ways:

Option A: Open HTML file directly

Open the following URL in your browser:

file:///Users/mac/storage/clawbot/arm_cpu_emulator_cpp/tools/viz_server/index.html

Then load the output JSON file through the file picker in the visualization interface.

Option B: Use the built-in web server

Start a local HTTP server that serves the visualization page automatically:

./build/arm_cpu_sim -f elf tests/data/test_elf_aarch64 -p 3000

Then open http://localhost:3000 in your browser. The visualization page loads automatically and the output data is available at the same address.

Verify the Build

Run the unit tests to confirm everything is working correctly:

# Run all unit tests
cd build && ctest --output-on-failure

# Or run the comprehensive test suite
./scripts/test_all.sh
