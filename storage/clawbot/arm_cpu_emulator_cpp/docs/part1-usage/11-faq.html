FAQ & Troubleshooting

Common issues and their solutions for building, running, and visualizing simulations with the ARM CPU Emulator.

Build Issues

Q: CMake configure fails with "CMake 3.20 or higher required"

A: Install a newer CMake: brew install cmake (macOS) or use your distribution's package manager.

Q: Compilation fails with C++20 errors

A: Ensure you have a C++20-capable compiler: clang 14+, GCC 12+, or Apple Clang 15+.

Q: Capstone fetch fails during CMake configure

A: Capstone v5 is fetched via CMake FetchContent. Ensure you have internet access during the first build. The fetched dependency is cached for subsequent builds.

Q: liblzma not found (ChampSim XZ support)

A: Install XZ utilities: brew install xz (macOS) or apt install liblzma-dev (Ubuntu). This is only needed for ChampSim XZ format.

Simulation Issues

Q: Simulation produces no output / empty JSON

A: Check that the trace file exists and is readable. Use -v for verbose output. Verify the format flag matches the file type.

Q: "Unsupported instruction" error during ELF simulation

A: The Capstone decoder supports ~500 ARM64 mnemonics. Some rare or privileged instructions may not be supported. Check the instruction at the reported PC address.

Q: Simulation runs forever / takes very long

A: Use -n (max instructions) or -c (max cycles) to limit simulation scope. Use -e (event mode) for faster batch simulation without visualization overhead.

Q: Different results each run

A: Results should be deterministic for the same input and configuration. If you see non-deterministic behavior, check for uninitialized memory in your test program.

Visualization Issues

Q: Visualization page is blank / no data shown

A: Ensure you loaded the correct output.json file. The file must be generated by a simulation run (not event mode -e, which doesn't produce Konata data). Check browser console for JavaScript errors.

Q: Web server mode (-p) doesn't work

A: Ensure the port is not in use. Try a different port: -p 8080. Check firewall settings.

Cross-Compilation Issues

Q: aarch64-elf-gcc not found

A: Install via Homebrew: brew install aarch64-elf-gcc (macOS). On Linux: apt install gcc-aarch64-linux-gnu.

Q: Dynamically linked ELF fails to load

A: Dynamic ELF requires a glibc sysroot for the target architecture. Use static linking (-static -nostdlib) for reliable simulation.

Performance Issues

Q: Simulation is slow

A: Try these optimizations:

  • Use -e (event mode) to skip visualization overhead
  • Reduce max instructions with -n
  • Use --preset minimal for faster testing
  • Use Release build: ./scripts/build.sh Release

Q: High memory usage

A: Memory usage scales with instruction window size and trace length. Reduce --window-size or use -n to limit instructions.

Still stuck?

Check the Automation Scripts page for the full test suite (./scripts/test_all.sh), which can help identify whether an issue is environment-specific or a bug in the simulator.
