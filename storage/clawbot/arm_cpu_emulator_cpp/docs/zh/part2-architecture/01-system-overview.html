系统概览

ARM CPU 仿真器的高层架构：面向 ARMv8 乱序处理器的 ESL 时序仿真器。

ARM CPU 仿真器是一个面向 ARMv8 乱序处理器的 ESL（Electronic System Level，电子系统级）时序仿真器。使用 C++20 编写，从原始 Rust 实现移植而来，可仿真指令流水线、Cache 层次结构和 CHI 互连协议。

模块架构

系统由多个互连的子系统组成：

子系统 头文件 职责
OoO Engine ooo/ooo_engine.hpp 指令窗口、依赖跟踪、重排序缓冲
Memory memory/memory_subsystem.hpp L1/L2/L3 Cache、LSQ、DDR 控制器
CHI chi/chi_manager.hpp AMBA CHI 协议（RN-F、HN-F、SN-F 节点）
Decoder decoder/capstone_decoder.hpp Capstone v5 ARM64 反汇编（约 500 条助记符）
Input input/instruction_source.hpp 工厂模式，分派至格式特定的解析器
Stats stats/stats_collector.hpp IPC、CPI、Cache 命中率、MPKI、指令混合比
Visualization visualization/visualization_state.hpp Konata JSON 导出，供流水线时间线查看器使用
Analysis analysis/topdown.hpp TopDown 微架构分析方法

两条仿真路径

仿真器提供两种仿真模式，共享通用子系统：

  1. CPUEmulator（cpu.hpp）：完整路径，包含 CHI 互连、Konata 可视化导出和 trace 输出。这是默认模式。
  2. SimulationEngine（simulation/simulation_engine.hpp）：基于事件的路径，向可插拔的 sink 发射 SimulationEvent。无 Konata 导出。使用 -e 参数启用。

两条路径共享相同的 OoO Engine、Memory Subsystem 和 Stats Collector。

数据流

指令在系统中的流动路径如下：

InstructionSource（trace/ELF 输入）
  → Fetch/Dispatch
  → OoOEngine（Rename, Issue, Execute）
  → MemorySubsystem（LSQ, L1/L2/L3, DDR）
  → Commit
  → Stats/Visualization

├─ cpu.cpp              # CPUEmulator 实现

├─ decoder/             # 解码器实现
├─ ooo/                 # OoO Engine 实现
