类型系统与数据结构

ARM CPU 仿真器中使用的核心类型、枚举和数据结构，包括寄存器定义、指令表示和错误处理。

所有核心类型定义在 include/arm_cpu/types.hpp（732 行）。该文件使用现代 C++20 特性，包括作用域枚举、结构体、constexpr 和 concept。

寄存器类型

仿真器使用强类型枚举建模 ARM64 寄存器文件：

// 通用寄存器
enum class Reg : uint8_t { X0, X1, ..., X28, X29, FP, LR, SP, XZR };

// 向量/SIMD 寄存器
enum class VReg : uint8_t { V0, V1, ..., V31 };

// 谓词寄存器 (SVE)
enum class PReg : uint8_t { P0, P1, ..., P15 };
  • Reg：31 个通用寄存器（X0–X30）加上帧指针（FP/X29）、链接寄存器（LR/X30）、栈指针（SP）和零寄存器（XZR）。
  • VReg：32 个 128 位 SIMD/NEON 寄存器（V0–V31）。
  • PReg：16 个用于 SVE 的谓词寄存器（P0–P15）。

OpcodeType 枚举（约 80 个条目）

对仿真器识别的每条指令进行分类的核心枚举。按功能类别分组：

enum class OpcodeType : uint8_t {
    // 整数 ALU
    Add, Sub, And, Or, Xor, Shift, Rotate, Neg, Not,
    Mul, Mneg, Div, Udiv, Rem, Urem,
    // 比较
    Cmp, Cmn, Tst,
    // 数据移动
    Mov, Mvn, Movz, Movk,
    // Load/Store
    Load, Store, LoadPair, StorePair, LoadByte, StoreByte,
    LoadHalf, StoreHalf, LoadSigned, Prefetch,
    // 分支
    Branch, BranchLink, BranchReg, CondBranch,
    CompareBranch, TestBranch, Return, ExceptionReturn,
    // 浮点
    FpAdd, FpSub, FpMul, FpDiv, FpSqrt,
    FpCmp, FpConvert, FpRound,
    // SIMD/NEON
    SimdAdd, SimdSub, SimdMul, SimdShift, SimdCompare,
    SimdLoad, SimdStore, SimdMove,
    // SVE
    SveAdd, SveSub, SveMul, SveLoad, SveStore,
    // SME
    SmeAdd, SmeMul, SmeLoad, SmeStore, SmeOuterProduct,
    // 加密
    AesEncode, AesDecode, AesMix, AesMc,
    Sha1Hash, Sha256Hash, Sha512Hash,
    // 系统
    Barrier, Nop, Sys, Msr, Mrs,
    // 其他
    Unknown, Count
};

分类函数

工具函数将操作码分为不同类别，供 Pipeline 调度器和统计收集器使用：

bool is_memory_op(OpcodeType);   // Load, Store, LoadPair, StorePair 等
bool is_branch(OpcodeType);      // Branch, BranchLink, CondBranch 等
bool is_compute(OpcodeType);     // Add, Sub, Mul 等
bool is_simd(OpcodeType);        // SimdAdd, SimdSub 等
bool is_sve(OpcodeType);         // SveAdd, SveSub 等
bool is_sme(OpcodeType);         // SmeAdd, SmeMul 等
bool is_crypto(OpcodeType);      // AesEncode, Sha1Hash 等
bool is_barrier(OpcodeType);     // Barrier

Instruction 结构体（Builder 模式）

表示仿真 Pipeline 中单条指令的核心数据结构。使用 Builder 模式方便构造：

struct Instruction {
    uint64_t pc = 0;
    OpcodeType opcode = OpcodeType::Unknown;
    std::string mnemonic;

    // 寄存器
    StaticVector<Reg, 4> src_regs;
    StaticVector<Reg, 4> dst_regs;
    StaticVector<VReg, 4> src_vregs;
    StaticVector<VReg, 4> dst_vregs;

    // 内存
    uint64_t mem_addr = 0;
    std::size_t mem_size = 0;
    bool is_load = false;
    bool is_store = false;

    // 分支
    uint64_t branch_target = 0;
    bool branch_taken = false;

    // Builder 方法
    Instruction& with_pc(uint64_t pc);
    Instruction& with_opcode(OpcodeType op);
    Instruction& with_src_reg(Reg r);
    Instruction& with_dst_reg(Reg r);
    Instruction& with_mem_access(uint64_t addr, std::size_t size, bool is_load);
    Instruction& with_branch(uint64_t target, bool taken);
};

InstrStatus 枚举

跟踪指令在乱序 Pipeline 中的生命周期：

enum class InstrStatus : uint8_t {
    Waiting,     // 在窗口中，等待依赖
    Ready,       // 依赖已解决，准备发射
    Executing,   // 正在功能单元上执行
    Completed,   // 执行完成，结果可用
    Committed    // 已从 Pipeline 退休
};

StaticVector<T, N>

一种栈分配的小型向量，避免堆分配。在寄存器列表和依赖集等场景中广泛使用：

  • 固定容量 N：最大元素数量（编译时常量）
  • 动态大小可达 N：可以容纳 0 到 N 个元素
  • 无堆分配：所有存储在栈上，适合热路径数据结构
  • 典型用法：StaticVector<Reg, 4> 用于寄存器操作数列表

错误处理：Result<T>

仿真器使用基于 std::variant 的 Rust 风格错误处理模式：

// Rust 风格错误处理
template<typename T>
using Result = std::variant<T, EmulatorError>;

// 成功辅助函数
template<typename T>
Result<T> Ok(T value);

// 错误辅助函数
template<typename T>
Result<T> Err(EmulatorError error);

// TRY 宏：遇到错误时提前返回
#define TRY(expr) \
    ({ auto _result = (expr); \
       if (std::holds_alternative<EmulatorError>(_result)) \
           return std::get<EmulatorError>(_result); \
       std::get<T>(std::move(_result)); })

此模式在不使用异常的情况下沿调用栈传播错误，遵循原始 Rust 实现中使用的相同约定。TRY 宏解包成功值或在遇到错误时立即返回，类似于 Rust 的 ? 操作符。
