统计收集系统

两大核心组件 — StatsCollector 负责原始事件追踪，PerformanceMetrics 负责派生分析 — 提供全面的仿真洞察。

StatsCollector

源文件：stats/stats_collector.hpp

StatsCollector 是仿真过程中的核心统计收集组件。它每个周期由 CPU 流水线更新，记录以下类别的事件：

  • 每条指令的延迟 — 每条指令从 Fetch 到 Commit 的周期数
  • Cache 事件 — 每个 Cache 级别（L1、L2、L3）的命中、未命中和驱逐次数
  • 内存操作计数 — 执行的 Load 和 Store 总数
  • 分支预测结果 — 正确和错误的预测次数
  • Pipeline 停顿周期 — 按类型（数据冒险、Cache 未命中、结构冒险等）分类的停顿周期明细
  • 功能单元使用情况 — 每种执行单元类型的周期计数

关键记录方法：

record_cache_access()
记录 Cache 访问事件，包含命中/未命中结果、延迟和 Cache 级别。
record_branch()
记录分支预测事件，包含预测结果与实际结果。
record_instruction()
记录指令完成，包含每条指令的时序数据。

PerformanceMetrics

源文件：stats/performance_metrics.hpp

PerformanceMetrics 组件从 StatsCollector 收集的原始统计数据计算派生指标。PerformanceStats 结构体包含：

指标 描述
total_instructions 仿真的指令总数
total_cycles 仿真总周期数
ipc 每周期指令数（total_instructions / total_cycles）
cpi 每指令周期数（total_cycles / total_instructions）
Cache 统计 每个级别：命中次数、未命中次数、命中率、MPKI（每千条指令的未命中次数）
分支预测 正确预测次数、总预测次数、准确率百分比
功能单元利用率 每个单元处于忙碌状态的周期占比
Pipeline 停顿分布 按停顿类型分类的停顿周期明细
指令混合 按 OpcodeType 类别分类的明细

指标组件还维护两个时间序列用于趋势分析：

  • IPC 时间序列 — 每 1000 个周期采样一次 IPC，用于 IPC 趋势可视化
  • 实际运行时间序列 — 跟踪实时仿真速度（每实际秒执行的指令数）

指令混合类别

统计数据按 OpcodeType 分为以下类别：

类别 操作码
整数 ALU Add, Sub, And, Or, Xor, Shift, Move 等
整数乘法 Mul, Mneg, MADD, MSUB 等
Load Ldr, Ldp, Ldrb, Ldrh, Ldrsb, Ldrsw, Ldur, Prfm 等
Store Str, Stp, Strb, Strh, Stur 等
分支 B, Bl, Cbz, Cbnz, B.cond, Br, Blr, Ret 等
FP Fadd, Fsub, Fmul, Fdiv, Fcmp, Fmov, Fcvt 等
SIMD SimdAdd, SimdMul, SimdDup, SimdIns, SimdZip 等
系统 Barrier, Nop, Mrs, Msr, Sys 等
加密 Aes*, Sha* 等
SVE SveAdd, SveMul, SveCmp, SveWhile 等
SME Sme* 操作（Streaming SVE 模式）
指令混合与性能关联

指令混合对于理解仿真瓶颈非常有价值。例如，Load 指令占比高且 L1 命中率低，说明是内存受限的工作负载。分支比例高且预测准确率低，则表明是控制流受限的行为。

Trace 输出

可以通过 --save-trace 标志启用详细的逐指令执行 Trace。Trace 记录：

  • 周期时间戳，对应每个 Pipeline 阶段（Fetch、Decode、Rename、Dispatch、Issue、Execute、Writeback、Commit）
  • 寄存器值，每条指令读取和写入的寄存器值
  • 内存地址，Load/Store 指令访问的内存地址
  • 分支预测与结果，条件分支的预测和实际跳转结果
--save-trace
在输出 JSON 文件中启用详细的逐指令执行 Trace 记录。
--dump-range <start>:<end>
仅导出指定范围内（含边界）指令的 Pipeline Trace。示例：--dump-range 100:200。
Trace 文件大小

启用 --save-trace 可能产生较大的输出文件，尤其是长时间仿真时。使用 --dump-range 将 Trace 输出限制在感兴趣的指令窗口内，或配合 -n 限制总指令数。
