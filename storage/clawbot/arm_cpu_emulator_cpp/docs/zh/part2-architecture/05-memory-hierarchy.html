内存层次结构

三级 Cache 层次结构配合 DDR 内存控制器，支持可配置的大小、关联度和延迟。

Cache 组织结构

仿真器建模了一个真实的三级 Cache 层次结构，由 DDR 内存控制器支撑。每一级均可通过命令行参数或预设独立配置。

级别 默认大小 关联度 行大小 命中延迟
L1 64 KB 4 路 64 B 4 个周期
L2 512 KB 8 路 64 B 12 个周期
L3 8 MB 16 路 64 B 40 个周期
DDR — — — 150 个周期（基础）

Cache 实现

源文件：memory/cache.hpp、memory/cache.cpp

Cache 实现为组相联结构，具有以下特性：

  • LRU（Least Recently Used）替换策略 — 驱逐每组中最近最少访问的 Cache 行
  • 组数计算方式：cache_size / (associativity × line_size)
  • CacheAccessInfo 结构体跟踪每次内存操作的访问类型（命中/未命中）、延迟和 Cache 级别
  • 写回策略用于 Store — 修改后的数据仅在驱逐时写回下一级
  • Cache-to-Cache 传输在 L3 中建模，用于多核一致性场景

DDR 内存控制器

DDR 内存控制器建模了真实的 DRAM 时序参数：

参数 值
基础延迟 150 个周期
行缓冲命中奖励 -30 个周期（总计 120 个周期）
Bank 冲突惩罚 +20 个周期
Bank 数量 8 个 Bank，用于并行访问

控制器建模了真实的 DRAM 时序，包括激活、读/写和预充电操作。行缓冲命中（访问已打开的相同行）可获得延迟奖励，而 Bank 冲突（访问已打开 Bank 中的不同行）则会产生惩罚。

Load/Store Queue (LSQ)

源文件：memory/lsq.hpp

Load/Store Queue 位于执行单元和 Cache 层次结构之间，管理内存排序和数据转发：

参数 默认值
队列大小 64 个条目
Load 流水线数 2（可配置）
Store 流水线数 1（可配置）
未完成内存请求数 16（可配置）

每个 LSQEntry 结构跟踪以下信息：

  • 虚拟地址和物理地址
  • 指令指针（PC）
  • 数据大小（字节宽度）
  • 完成状态
  • 对前序 Store 的依赖（用于 Load-Store 转发）

LSQ 具有独立的 Load 和 Store 流水线，允许 Load 和 Store 独立发射。当较新的 Load 依赖于队列中较旧的 Store 时，支持 Store-to-Load 转发。

内存子系统集成

源文件：memory/memory_subsystem.hpp

MemorySubsystem 类将所有内存组件组合为统一接口：

  • 将 L1、L2、L3 Cache 和 DDR 控制器组合为单一访问路径
  • 启用 CHI 协议时处理 Cache 一致性
  • CPU 为每个内存操作调用 record_cache_access()
  • 返回包含完整计时信息的 CacheAccessInfo 结构体

Cache 访问流程

内存访问从 L1 向下遍历至 DDR，每次未命中累加延迟：

CPU Load/Store
  → L1 Cache（命中 4 个周期）
  → L2 Cache（命中 12 个周期，+4 L1 未命中）
  → L3 Cache（命中 40 个周期，+12 L2 未命中）
  → DDR（基础 150 个周期，+40 L3 未命中）

在所有 Cache 级别全部未命中时，总访问延迟为 4 + 12 + 40 + 150 = 206 个周期。这一惩罚促使 OoO Engine 在内存操作未完成时执行独立指令以隐藏内存延迟。

非阻塞 Cache

Cache 未命中不会停顿流水线。LSQ 跟踪未完成的内存请求，并在数据返回时唤醒依赖指令。这对于在内存密集型工作负载上实现良好的 IPC 至关重要。

配置参数

--l1-size <KB>
L1 数据 Cache 大小（千字节）。默认：64。常用值：32、64、128。
--l2-size <KB>
L2 Cache 大小（千字节）。默认：512。常用值：256、512、1024。
--l3-size <KB>
L3（末级）Cache 大小（千字节）。默认：8192（8 MB）。常用值：4096、8192、16384、32768。
