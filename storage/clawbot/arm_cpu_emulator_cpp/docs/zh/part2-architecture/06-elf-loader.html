ELF 加载器

解析 ELF64 小端序 ARM64 二进制文件，将段映射到仿真内存并解析符号。

源文件：elf/elf_loader.hpp、elf/elf_loader.cpp

ELF 解析

ELF 加载器执行三个阶段的解析：

  • ELF Header — 验证魔数（\x7fELF）、类别（64 位）、字节序（小端序）和机器类型（AArch64）。不符合条件的二进制文件将被拒绝。
  • Program Headers — 识别需要映射到仿真内存的 PT_LOAD 段。每个 Program Header 指定段的虚拟地址、文件偏移和大小。
  • Section Headers — 读取节元数据，用于符号解析、字符串表和调试信息。

段加载

PT_LOAD 段被加载到仿真内存中，权限处理如下：

标志 常量 段类型
PF_X 0x1 可执行（代码段）
PF_W 0x2 可写（数据段）
PF_R 0x4 可读（所有段）

ELF 文件中的虚拟地址直接映射到仿真器的内存空间。总内存大小通过 -m 参数配置（默认：256 MB）。

内存大小配置

使用 -m <MB> 设置仿真内存大小。如果程序访问较大的内存区域，请增大此值。例如：-m 1024 表示 1 GB 仿真内存。

符号解析

加载器解析符号表以支持名称查找和地址到名称的映射：

  • 解析 .symtab（静态）和 .dynsym（动态）节
  • 支持 STT_FUNC（函数符号）和 STT_OBJECT（数据符号）
  • C++ 名称反修饰，用于修饰后的符号（例如 _Z3foov → foo()）
  • PLT（Procedure Linkage Table）符号检测，用于动态链接的函数
  • 按名称和按地址进行符号查找

数据结构

加载器使用以下核心数据结构：

struct MemorySegment {
    uint64_t vaddr;       // 虚拟地址
    uint64_t size;        // 段大小
    uint64_t flags;       // 权限（R/W/X）
    std::vector<uint8_t> data;  // 段内容
};

struct SymbolTable {
    struct Symbol {
        std::string name;
        uint64_t address;
        uint64_t size;
        uint8_t type;      // STT_FUNC, STT_OBJECT, 等
        uint8_t binding;   // STB_LOCAL, STB_GLOBAL, 等
    };
    std::vector<Symbol> symbols;
};

入口点检测

仿真器使用基于优先级的方法确定初始程序计数器（PC）：

  1. ELF 入口点字段 — 使用 ELF Header 中的 e_entry 字段作为主要来源
  2. _start 符号后备 — 如果入口点为 0，则在符号表中搜索 _start 符号
  3. 动态 ELF 支持 — 对于动态链接的二进制文件，支持 GOT/PLT 解析以找到实际入口点

功能仿真（FunctionalSim）

源文件：input/functional_sim.hpp、input/functional_sim.cpp

FunctionalSim 组件是一项关键创新，能够实现精确的时序仿真：

  • 预执行 ELF 二进制文件以确定实际的分支结果
  • 为主时序仿真器提供"完美分支预测"
  • 功能性地执行指令（无时序模型），记录每条分支指令的跳转/不跳转结果
  • 分支结果被馈送到主时序仿真器，从仿真中消除误预测惩罚
为什么需要功能预仿真？

通过先功能性地运行二进制文件，仿真器在时序仿真开始之前就知道了每条分支的确切结果。这将内存层次结构和流水线效果与分支预测质量隔离开来，使得更容易独立研究微架构行为。
