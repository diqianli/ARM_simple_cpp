输入源接口

可插拔的输入源架构，支持多种 Trace 格式，通过工厂函数分派到合适的解析器。

仿真器使用可插拔的输入源架构。工厂函数 create_source() 根据指定的格式分派到相应的解析器，允许在核心仿真循环几乎不变的情况下添加新的输入类型。

InstructionSource 接口

定义在 input/instruction_source.hpp，此抽象基类定义了所有输入源的契约：

  • has_next()：检查 Trace 中是否还有更多指令
  • next()：获取 Trace 中的下一条指令
  • peek()：查看下一条指令但不消费它
  • reset()：重置到 Trace 的开头

输入源实现

1. TraceInput（input/instruction_source.cpp）

从人类可读的文本 Trace 文件中读取指令。逐行解析，格式为 OPCODE [dest,] src1[, src2]。

  • 格式标志：-f text
  • 覆盖范围：约 40 个助记符（整数 ALU、Load/Store、分支、基础 FP）
  • 使用场景：快速调试和手动 Trace 创建

2. ElfTrace（input/elf_trace.hpp、input/elf_trace.cpp）

通过运行时反汇编代码段来执行 ELF 二进制文件。这是获得完整 ISA 覆盖的推荐输入源。

  • 格式标志：-f elf
  • 解码器：使用 Capstone 解码器进行指令解码（约 500 个 ARM64 助记符）
  • 分支预测：通过 FunctionalSim 实现功能性预仿真，用于准确的分支目标解析
  • 循环重放：支持长时间运行基准测试的循环重放

3. FunctionalSim（input/functional_sim.hpp、input/functional_sim.cpp）

功能性地（不进行时序仿真）预执行 ELF 二进制文件，以确定实际的分支结果。这为 ElfTrace 源提供"完美分支预测"数据。

  • 角色：在时序仿真运行之前记录实际的分支结果
  • 输出：生成具有正确分支目标的指令 Trace
  • 使用者：ElfTrace 内部使用它进行分支预测

4. ChampSimTrace（input/champsim_trace.cpp）

解析 ChampSim 学术 Trace 格式，这是计算机架构研究中常用的格式。

  • 格式标志：-f champsim 或 -f champsim_xz
  • 压缩：支持原始和 XZ 压缩的 Trace（通过 liblzma）
  • 格式：每行 0 addr PCS accesses

工厂函数

工厂函数根据配置创建适当的输入源：

std::unique_ptr<InstructionSource> create_source(
    const TraceInputConfig& config,
    const CPUConfig& cpu_config);

它根据 config.format（一个 TraceFormat 枚举）进行分派，为指定的输入类型选择正确的解析器和解码器组合。

ELF 循环重放

对于在紧密循环中运行的基准测试，仿真器可以多次重放 ELF，以防止 Trace 过早结束。这通过最大指令数（-n）或最大周期数（-c）限制来配置，它们控制仿真在 Trace 长度无关的情况下何时终止。
