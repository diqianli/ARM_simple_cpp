指令解码器

将 ARM64 指令编码转换为内部结构，供流水线执行使用。

解码器子系统将指令编码转换为内部 Instruction 结构。提供两条解码路径，根据输入格式自动选择。

解码器选择

输入格式 解码器 ISA 覆盖范围
ELF CapstoneDecoder 约 500 条助记符
Text TextTraceParser 约 40 条助记符
ChampSim ChampSimTrace ChampSim 格式
JSON JsonTrace 不适用

Capstone 解码器

主解码器（decoder/capstone_decoder.hpp）封装 Capstone v5，提供完整的 ARM64 反汇编功能：

  • 封装 Capstone v5 反汇编引擎，支持 ARM64
  • 覆盖 ARMv8.0 至 ARMv8.7-A
  • 支持扩展：SVE、SME、Crypto（AES、SHA1、SHA2、SHA3）
  • 通过 MnemonicLookup 表实现约 500 条助记符映射
  • 检测条件分支以支持分支预测

关键流程：

  1. Capstone 反汇编指令字节
  2. 助记符字符串映射到内部 OpcodeType 枚举
  3. 解析操作数（寄存器、立即数、内存地址）
  4. 使用 Builder 模式构建 Instruction

自定义 AArch64 解码器

轻量级后备解码器（decoder/aarch64_decoder.hpp），在 Capstone 不可用时使用：

  • 子解码器：arithmetic、logical、load_store、branch、simd_neon、fp、crypto、system
  • 在 Capstone 不可用时作为后备方案
  • 使用 Builder 模式的 DecodedInstruction 结构体

指令 Builder 模式

指令使用流畅的 Builder 模式构建：

Instruction instr;
instr.with_opcode(OpcodeType::Add)
     .with_pc(pc)
     .with_src_reg(Reg::X1)
     .with_src_reg(Reg::X2)
     .with_dst_reg(Reg::X0);

条件分支检测

Capstone 解码器识别条件分支（B.cond、CBZ、CBNZ、TBZ、TBNZ）并设置分支预测元数据供流水线使用。此信息流入 OoO Engine，以支持对预测跳转路径的推测执行。
