乱序执行引擎

仿真器的核心组件，实现支持推测执行的乱序执行机制。

OoO Engine 是仿真器的核心，实现了支持推测执行的乱序执行机制。

架构概览

三个主要组件协同工作以实现乱序执行：

  1. 指令窗口（ooo/window.hpp）：Reorder Buffer（ROB），保存所有在飞指令
  2. 依赖跟踪器（ooo/dependency.hpp）：跟踪寄存器和内存依赖
  3. 调度器（ooo/scheduler.hpp）：选择就绪指令进行执行

指令窗口（ROB）

指令窗口是 OoO Engine 的核心数据结构：

  • 大小通过 window_size 配置（默认：128）
  • 每个条目是一个 WindowEntry，包含：
    • Instruction 数据
    • InstrStatus 状态机：Waiting → Ready → Executing → Completed → Committed
    • 物理寄存器映射
    • 依赖计数器（剩余寄存器 + 内存依赖数）
  • 指令从尾部进入，从头部提交（按序提交）
  • 窗口满时 Fetch 停顿

依赖跟踪

依赖在两个层面进行跟踪：

寄存器依赖

  • 通过寄存器重命名建立生产者-消费者关系
  • 每条指令跟踪其产生的物理寄存器（目标）和消费的物理寄存器（源）
  • 当生产者完成 Writeback 时解除依赖

内存依赖

  • Load-after-Store (RAW)：Load 必须等待对相同地址的前序 Store
  • Store-after-Store (WAW)：Store 必须按程序顺序完成
  • 基于地址的消歧，用于 Load/Store 排序

调度器

调度器每个周期检查窗口中的所有指令并执行以下步骤：

  1. 检查窗口中所有处于 Waiting 状态的指令
  2. 唤醒依赖已满足的指令（转换到 Ready 状态）
  3. 每个周期最多选择 issue_width 条 Ready 指令
  4. 优先级：最旧的就绪指令优先（按程序顺序）

执行单元类型

单元类型 说明 典型延迟
IntAlu 整数算术/逻辑 1 个周期
IntMul 整数乘法 3 个周期
Load 内存加载 4 个周期（L1 命中）
Store 内存存储 1 个周期
Branch 条件/无条件分支 1 个周期
FpSimd 浮点 / SIMD 3–5 个周期
System 系统指令（屏障等） 1–10 个周期
Crypto 加密指令 可变
