可视化系统

生成兼容 Konata 的 JSON 格式 Pipeline 时间线可视化数据，并提供基于 Web 的交互式渲染界面。

VisualizationState

源文件：visualization/visualization_state.hpp

VisualizationState 类是所有可视化数据的中央状态管理器。它维护一个 VisualInstruction 条目列表，每个条目包含：

  • 指令元数据 — PC、操作码、助记符和操作数字符串
  • Pipeline 阶段时间戳 — 指令进入每个阶段的周期号（fetch、decode、rename、dispatch、issue、execute、memory、writeback、commit）
  • 颜色编码 — 基于指令类型，用于时间线中的视觉区分

状态管理器还提供：

  • 快照管理，用于大型 Trace 的增量渲染
  • 历史记录，用于可视化界面中的撤销/重做导航

PipelineTracker

源文件：simulation/pipeline_tracker.hpp

PipelineTracker 在仿真过程中记录阶段转换。它使用 StageTiming 结构体存储指令进入每个 Pipeline 阶段的周期号。Tracker 在每个阶段转换时由 CPU 流水线调用：

方法 Pipeline 阶段
record_fetch()取指
record_decode()译码
record_rename()寄存器重命名
record_dispatch()分派 / 分配
record_issue()发射 / 调度
record_execute()执行
record_memory()内存访问
record_writeback()写回
record_commit()提交 / 退休

PipelineTrackerViz

源文件：visualization/pipeline_tracker_viz.hpp

PipelineTrackerViz 组件将 Pipeline 时间线渲染为交互式 HTML/Canvas 可视化。指令按类型进行颜色编码：

指令类型 颜色 十六进制代码
ALU蓝色#3b82f6
Load绿色#22c55e
Store橙色#f97316
Branch红色#ef4444
FP / SIMD紫色#a855f7
System灰色#6b7280
Crypto黄色#eab308

Konata JSON 导出

可视化系统生成的 JSON 输出格式兼容 Konata Pipeline 查看器。数据结构按周期组织，每个周期包含各个 Pipeline 阶段中存在的指令：

{
  "core": [
    {
      "cycle": 0,
      "fetch": [{"pc": 0x1000, "instr": "ADD X0, X1, X2"}],
      "decode": [...],
      "rename": [...],
      "dispatch": [...],
      "issue": [...],
      "execute": [...],
      "memory": [...],
      "writeback": [...],
      "commit": [...]
    }
  ]
}

每个指令条目包含程序计数器（PC）和反汇编指令字符串，使查看器能够为每个 Pipeline 槽位同时显示地址和助记符信息。

Web 服务器

可视化 Web 界面位于 tools/viz_server/：

文件 用途
index.html 主可视化页面，包含文件选择器和多标签视图
app_static.js JavaScript 应用逻辑，用于数据加载和视图管理
style.css 可视化专用样式（时间线、指标面板、控件）
konata/konata_renderer.js Konata Pipeline 时间线渲染器，支持缩放和平移
perf_stats_view.js 表格化性能指标视图（IPC、Cache 统计、指令混合）

可视化可以通过两种方式访问：

  • 内置 Web 服务器：使用 -p <port> 启动 HTTP 服务器，自动在 http://localhost:<port> 提供可视化页面
  • 静态文件：直接在浏览器中打开 tools/viz_server/index.html，通过文件选择器加载输出 JSON 文件

Kanata 日志导出

源文件：visualization/kanata_log_exporter.hpp

KanataLogExporter 提供执行 Trace 的替代导出格式：

  • 以 Kanata 日志格式导出执行 Trace
  • 兼容外部 Kanata 可视化工具
  • 包含每条指令的阶段时序和依赖信息
  • 适用于跨工具分析和与其他仿真器输出的比较
