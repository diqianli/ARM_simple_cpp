配置预设

适用于常见仿真场景的内置 CPU 微架构预设，从均衡通用配置到 GEM5 交叉验证。

概览

CPUConfig 为常见使用场景提供了多种预设。通过 CLI 选择预设：

./build/arm_cpu_sim --preset <name> -f elf program.elf

可用预设：default、high_performance、minimal、gem5_o3_arm_v7a。

预设与参数覆盖混用

选择预设后，任何参数都可以通过 CLI 标志单独覆盖。例如：--preset gem5_o3_arm_v7a --window-size 64 从 GEM5 预设出发，但将 ROB 大小改为 64。

default_config()

均衡配置，适用于一般仿真。这是未指定 --preset 标志时使用的默认预设。

参数 值
window_size (ROB)128
fetch_width8
issue_width4
commit_width4
lsq_size64
load_pipeline_count2
store_pipeline_count1
outstanding_requests16
L1 Cache64 KB, 4-way, 64B line, 4 周期延迟
L2 Cache512 KB, 8-way, 64B line, 12 周期延迟
L3 Cache8 MB, 16-way, 64B line, 40 周期延迟
DDR 基础延迟150 周期
频率2000 MHz

high_performance()

激进型乱序配置，用于建模高性能核心。具有更大的指令窗口、更宽的 Issue/Commit 和更多的 Load/Store Pipeline。

参数 值
window_size256
issue_width6
commit_width6
lsq_size128
load_pipeline_count4
store_pipeline_count2
outstanding_requests32

未列出的 Cache 和内存参数继承自 default_config()。

minimal()

小型配置，用于快速仿真或测试。适用于开发期间的快速迭代，或在小程序上验证仿真器正确性。

参数 值
window_size16
issue_width2
commit_width2
lsq_size8
load_pipeline_count1
store_pipeline_count1
outstanding_requests4
L1 Cache4 KB, 2-way
L2 Cache16 KB, 2-way

未列出的 L3 Cache 和内存参数继承自 default_config()。

gem5_o3_arm_v7a()

匹配 GEM5 的 O3_ARM_v7a 默认配置，用于交叉验证。此预设支持我们的仿真器与 GEM5 之间的仿真结果直接对比。

参数 值 GEM5 对应项
window_size40ROB size
fetch_width3fetchWidth
issue_width8issueWidth
commit_width8commitWidth
lsq_size32LQ(16) + SQ(16)
L1 Cache32 KB, 2-way, 2 周期ICache / DCache
L2 Cache1 MB, 16-way, 20 周期L2
GEM5 O3_ARM_v7a 中没有 L3

GEM5 的 O3_ARM_v7a 配置不包含 L3 Cache。使用此预设时，我们的仿真器保留默认 L3（8 MB, 16-way）。要禁用 L3 以获得更接近的匹配，请使用 --l3-size 0 覆盖。

DDR 内存控制器

DDR 内存控制器设置在所有预设中共享，除非显式覆盖：

设置 值 描述
基础延迟150 周期内存请求的默认访问延迟
行缓冲命中奖励-30 周期请求的行已打开时的延迟减少量
Bank 冲突惩罚+20 周期两个请求访问同一 Bank 时的额外延迟
Bank 数量8独立的 DRAM Bank 数量
覆盖 DDR 设置

DDR 参数可以像其他配置选项一样通过 CLI 标志覆盖。例如，要建模更快的内存子系统：--ddr-base-latency 100 --num-banks 16。
