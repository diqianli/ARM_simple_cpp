可视化工具

使用 Konata Pipeline 时间线查看器检查指令级时序、Pipeline 停顿和功能单元利用率。

Konata Pipeline 查看器

仿真器内置了 Konata 风格的 Pipeline 时间线查看器，可将每条指令的 Pipeline 阶段时序渲染为可交互、可缩放的图表。

打开查看器

有两种方式访问可视化界面：

方式 A：直接打开 HTML 文件

file:///Users/mac/storage/clawbot/arm_cpu_emulator_cpp/tools/viz_server/index.html

方式 B：使用内置 Web 服务器

./build/arm_cpu_sim -f elf program.elf -p 3000

然后在浏览器中打开 http://localhost:3000。可视化页面会自动加载，输出数据在同一地址提供服务。

加载数据

使用可视化界面中的文件选择器加载仿真器生成的 output.json 文件。

时间线功能

Pipeline 时间线支持以下交互操作：

操作 方式
缩放 鼠标滚轮或缩放按钮
左右平移 在时间线上点击并拖动
搜索指令 按 PC 地址搜索以定位特定指令

按指令类型着色

指令按类型着色，便于快速视觉识别：

颜色 类别 示例
蓝色 ALU 操作 ADD, SUB, MUL, AND, ORR, EOR
绿色 Load 操作 LDR, LDP, LDUR, LDRB, LDRH
橙色 Store 操作 STR, STP, STUR, STRB, STRH
红色 / 粉色 分支操作 B, BL, CBZ, CBNZ, TBZ, RET, BR
紫色 浮点 / SIMD FADD, FSUB, FMUL, FADD (vector), SIMD 操作

Pipeline 阶段

时间线显示每条指令在 Pipeline 各阶段的推进过程：

IF (Fetch) → DE (Decode) → RN (Rename) → DI (Dispatch) → IS (Issue) → EX (Execute) → ME (Memory) → WB (Writeback) → RR (Retire)
  • IF (Fetch) — 从指令流中取指
  • DE (Decode) — 指令解码和分类
  • RN (Rename) — 寄存器重命名以消除假依赖
  • DI (Dispatch) — 分发到 Issue 队列
  • IS (Issue) — 操作数就绪后 Issue 到功能单元
  • EX (Execute) — 在分配的功能单元上执行
  • ME (Memory) — 内存访问（仅 Load/Store 指令）
  • WB (Writeback) — 结果写回寄存器文件
  • RR (Retire) — 指令按程序顺序 Commit

性能指标面板

可视化页面包含一个指标面板，显示以下内容：

  • IPC — 整体每周期指令数
  • Cache 命中率 — L1、L2、L3 Cache 命中率
  • 分支预测准确率 — 正确预测分支的百分比
  • 功能单元利用率 — 各类型功能单元繁忙时间的分布

替代方案：Performance Stats View

如需表格形式的指标展示，perf_stats_view.js 组件提供了性能统计的结构化表格视图。当你需要直接比较数值而非查看时间线时，这非常有用。

Kanata 日志导出

仿真器可以 Kanata 格式导出执行 Trace，以便与其他支持 Kanata 日志格式的可视化工具配合使用。这适用于跨工具分析和与使用不同工具链的合作者共享结果。
