CLI 参数参考

ARM CPU 仿真器二进制文件的完整命令行接口参考。

用法语法

arm_cpu_sim [options] <trace_file>

trace 文件以位置参数的形式提供。所有其他参数以标志位的形式传递。参数的顺序可以任意排列。

输入控制

以下选项控制仿真器如何读取和处理输入指令。

-f, --format <type>
选择输入格式。支持的值：elf、text、champsim、champsim_xz、json。默认为 text。详见 输入格式。
<trace_file>
位置参数，指定输入文件的路径（ELF 二进制、text trace、ChampSim trace 或 JSON trace）。这是运行仿真唯一必需的参数。
-n, --max-instructions <N>
最大仿真指令数。值为 0（默认）表示仿真输入中的所有指令。适用于快速基准测试或限制仿真时间。
-s, --skip-instructions <N>
在开始仿真前跳过输入中的前 N 条指令。适用于跳过初始化代码或预热阶段。默认为 0。
-c, --max-cycles <N>
仿真器终止前的最大仿真周期数。作为安全限制防止无限循环。默认为 1000000（100 万周期）。

CPU 配置

以下选项配置仿真 CPU 流水线的微架构参数。

--window-size <N>
Reorder Buffer（ROB）/ 指令窗口大小。决定了同时处于飞行状态的指令数量。更大的值允许更多的指令级并行（ILP），但会增加资源开销。默认为 128。
--fetch-width <N>
每周期从指令流中取指的指令数。默认为 8。典型值范围从 4（顺序核心）到 8（宽超标量核心）。
--issue-width <N>
每周期发射到执行单元的最大指令数。默认为 4。这是调度器的峰值发射带宽。
--commit-width <N>
每周期从 ROB 提交的最大指令数。默认为 4。在实际配置中通常与发射宽度一致。
预设会覆盖单独设置

使用 --preset <name> 将覆盖所有单独指定的 CPU 和内存配置参数。建议先应用预设，再按需覆盖特定值。

内存配置

以下选项配置 Cache 层次结构和内存子系统时序。

--l1-size <KB>
L1 数据 Cache 大小（KB）。默认为 64 KB。L1 Cache 是距离 CPU 核心最近的 Cache，延迟最低。
--l2-size <KB>
L2 Cache 大小（KB）。默认为 512 KB。L2 Cache 作为 L1 和 L3 之间的中间层 Cache。
--l3-size <KB>
L3（最后一级）Cache 大小（KB）。默认为 8192 KB（8 MB）。L3 Cache 在多核配置中跨核心共享。
--frequency <MHz>
CPU 时钟频率（MHz）。默认为 2000 MHz（2 GHz）。该值用于计算时序指标和功耗估算。

输出控制

以下选项控制仿真器的输出内容和写入位置。

-o, --output <path>
输出 JSON 文件路径。该文件包含仿真统计信息、Pipeline 追踪数据和 Konata 可视化数据。默认为当前工作目录下的 output.json。
-j, --json-metrics
同时将性能指标摘要以 JSON 格式输出到标准输出（stdout）。适用于程序化处理和脚本管道。
--save-trace
启用输出文件中的详细执行追踪。包含逐指令时序信息和 Pipeline 阶段转换记录。
--dump-range <start>:<end>
仅导出指定范围（含边界）内指令的 Pipeline 追踪。格式为 start:end。例如 --dump-range 100:200 导出第 100 到第 200 条指令。适用于调试特定指令序列。

可视化

以下选项控制可视化和 Web 服务器功能。

-p, --port <port>
在指定端口启动内置 HTTP 服务器，提供 Konata 可视化界面。在浏览器中打开 http://localhost:<port> 查看 Pipeline 时间线。可视化页面会自动加载仿真输出数据。
-e, --event-mode
启用事件驱动（SimulationEngine）模式。在此模式下，仿真器将 SimulationEvent 对象发送到可插拔的 Sink，而非运行完整的 CPUEmulator 路径。此模式禁用 Konata 可视化导出，适用于分析场景。
--viz-format <fmt>
选择可视化输出格式。支持的值：konata（Konata Pipeline 时间线 JSON）、perf（性能统计格式）。默认格式为 Konata。

多实例仿真

以下选项启用并行运行多个独立仿真实例，适用于设计空间探索。

--multi-instance
启用多实例仿真模式。每个实例使用独立的配置和输入运行。需要指定 --num-instances 并提供独立的输入文件。
--num-instances <N>
要运行的仿真实例数量。默认为 4。仅在启用 --multi-instance 时有效。
--num-threads <N>
并行执行使用的线程数。默认根据可用 CPU 核心数自动检测。可以手动设置以获得更精细的控制。

预设配置

--preset <name>
应用命名配置预设，一次性设置 CPU 流水线、Cache 和内存参数。可用预设：
预设名称 说明
default 均衡配置：128 条目 ROB、8 宽取指、4 宽发射/提交、64 KB L1、512 KB L2、8 MB L3
high_performance 激进的乱序配置，具有更大的窗口和 Cache，追求最大 IPC
minimal 最小配置，结构精简，适用于快速仿真迭代
gem5_o3_arm_v7a 匹配 gem5 O3 ARM v7a CPU 模型的配置，用于跨仿真器对比
预设与单独参数的组合

预设值先被应用，然后显式指定的标志会覆盖对应的预设值。例如 --preset high_performance --l1-size 32 使用 high_performance 预设但将 L1 大小覆盖为 32 KB。

其他选项

-m, --memory-size <MB>
ELF 仿真的总模拟内存大小（MB）。默认为 256 MB。这是加载的 ELF 程序可用的模拟物理地址空间大小。如果程序需要访问更大的内存区域，请增加此值。
-t, --threads <N>
已弃用 请使用 --num-threads <N>。此标志保留用于向后兼容，但可能在未来的版本中移除。
-v, --verbose
启用详细输出。在仿真过程中打印额外信息，包括逐周期状态、Cache 事件和 Pipeline 阶段转换。适用于调试和理解仿真器行为。

示例命令

基本 ELF 仿真

./build/arm_cpu_sim -f elf tests/data/test_elf_aarch64

限制指令数并自定义输出

./build/arm_cpu_sim -f elf program.elf -n 10000 -o results/run1.json

高性能 CPU 配置

./build/arm_cpu_sim -f elf program.elf --preset high_performance -o results/hp.json

自定义 Pipeline 与可视化服务器

./build/arm_cpu_sim -f elf program.elf \
  --window-size 256 --issue-width 6 --commit-width 6 \
  --l1-size 128 --l2-size 1024 \
  -p 8080 -o results/custom.json

事件驱动模式与 JSON 指标

./build/arm_cpu_sim -e -f elf program.elf -j

调试特定指令范围

./build/arm_cpu_sim -f elf program.elf --dump-range 50:100 -v
