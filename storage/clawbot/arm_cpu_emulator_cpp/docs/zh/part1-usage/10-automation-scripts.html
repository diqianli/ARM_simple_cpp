自动化脚本

使用项目的自动化脚本套件进行构建、编译、测试和对比，专为开发和 CI 工作流设计。

1. scripts/build.sh — 构建脚本

一键构建，处理 CMake 配置和编译。

  • 用法：./scripts/build.sh [BuildType]
  • BuildType：Debug、Release（默认）、RelWithDebInfo、MinSizeRel
  • 通过 nproc / sysctl -n hw.ncpu 自动检测并行任务数
  • 输出二进制：build/arm_cpu_sim
./scripts/build.sh           # Release 构建
./scripts/build.sh Debug     # 带调试符号的 Debug 构建

2. scripts/compile_test_elf.sh — 交叉编译测试程序

将 scripts/test_elf_source.c 编译为 tests/data/test_elf_aarch64。

  • 自动检测交叉编译器：aarch64-elf-gcc、aarch64-linux-gnu-gcc、aarch64-none-elf-gcc
  • 用法：./scripts/compile_test_elf.sh [compiler]
  • 编译标志：-static -O2 -nostdlib
  • 使用 linux-gnu 编译器时还会编译动态链接变体
./scripts/compile_test_elf.sh                    # 自动检测
./scripts/compile_test_elf.sh aarch64-elf-gcc    # 指定编译器

3. scripts/test_all.sh — 完整测试套件

全面的 6 阶段测试流水线，用于验证整个仿真器。

  • 用法：./scripts/test_all.sh [--fast]
  • --fast 跳过压力测试

测试阶段

阶段 描述
1. BUILD CMake 配置 + 构建 + 二进制检查
2. UNIT TESTS ctest with output-on-failure
3. TEXT TRACE SIMULATION 基本 Trace、内存 Trace 和分支 Trace
4. ELF SIMULATION 静态和动态 ELF
5. MULTI-CONFIG Minimal、High-performance、Tiny L1、Large Cache 配置
6. STRESS 长 Trace（1M 周期）、内存 Trace（1000 条指令，500K 周期）

每个阶段独立运行 — 一个阶段的失败不会阻止后续阶段。结果以 PASS/FAIL/SKIP 报告，末尾有汇总。

./scripts/test_all.sh       # 完整测试套件
./scripts/test_all.sh --fast # 跳过压力测试

4. scripts/compare_gem5.py — GEM5 对比

将仿真器输出与 GEM5 O3 ARM 仿真器结果进行对比。

  • 使用 --preset gem5_o3_arm_v7a 以获得对齐的配置
  • 对比维度：周期数、指令数、IPC、执行时间
python3 scripts/compare_gem5.py --ours output.json --gem5 gem5_output.json

CI 集成

所有脚本都是自包含的，适用于 CI 流水线：

  • test_all.sh 成功时返回退出码 0，任何失败时返回 1
  • 结合 build.sh 实现完整的 CI 工作流
CI 工作流示例
# 在你的 CI 流水线中
./scripts/build.sh Release
./scripts/test_all.sh --fast
echo "CI exit code: $?"
