运行仿真

学习如何使用各种选项执行仿真、控制仿真范围以及运行并行多实例实验。

基本仿真

运行仿真器最简单的方式是传入一个 ELF 二进制文件：

# 输入文件：tests/data/test_elf_aarch64
# 输出文件：output.json（默认，当前目录下）
./build/arm_cpu_sim -f elf tests/data/test_elf_aarch64

使用 -o 指定自定义输出路径：

# 输入文件：program.elf
# 输出文件：result.json
./build/arm_cpu_sim -f elf program.elf -o result.json

可视化服务器

启动内置 HTTP 服务器，提供 Konata Pipeline 可视化页面：

./build/arm_cpu_sim -f elf program.elf -p 3000

这会在端口 3000 上启动 HTTP 服务器。在浏览器中打开 http://localhost:3000 查看交互式 Pipeline 时间线。输出 JSON 数据与可视化页面一起提供。

事件驱动模式

使用 -e 标志通过 SimulationEngine 运行事件驱动模式。此模式跳过 Konata 可视化导出，适用于批处理或仅需指标的场合：

./build/arm_cpu_sim -e -f elf program.elf

限制仿真范围

使用以下标志控制仿真的指令数或周期数：

标志 说明 示例
-n 最大仿真指令数 -n 1000 — 仅仿真前 1000 条指令
-s 跳过前 N 条指令 -s 500 — 跳过前 500 条指令（预热）
-c 最大仿真周期数 -c 500000 — 在 500000 周期后停止
使用 -s 进行预热

使用 -s 跳过预热阶段（例如 -s 1000 -n 5000），使指标反映稳态行为而非冷启动效应。

多实例并行仿真

并行运行多个独立仿真实例，每个实例使用独立的配置。适用于设计空间探索：

./build/arm_cpu_sim -e --multi-instance --num-instances 4 --num-threads 4 -f elf program.elf
  • 每个实例接收独立的 CPUConfig
  • 结果写入独立的输出文件
  • 使用 --num-threads 控制并行度（默认为硬件并发数）
  • 结合 --preset 或单独的 --* 标志为每个实例变化参数

JSON 指标输出

-j 标志将紧凑的指标摘要以 JSON 格式输出到 stdout，独立于完整的输出文件：

./build/arm_cpu_sim -f elf program.elf -j

适用于脚本和自动化分析管道，便于快速获取 IPC、Cache 命中率和其他汇总指标。

组合选项

多个标志可以自由组合。例如：

# 输入文件：program.elf
# 输出文件：result.json
# 限制为 10000 条指令和 100000 个周期，同时在 stdout 输出 JSON 指标
./build/arm_cpu_sim -f elf program.elf -n 10000 -c 100000 -o result.json -j

典型工作流示例

1. 快速测试

以有限范围运行快速仿真以验证正确性：

./build/arm_cpu_sim -f elf tests/data/test_elf_aarch64 -n 100 -c 10000

2. 完整仿真与可视化

运行完整仿真并启动基于 Web 的可视化服务器：

./build/arm_cpu_sim -f elf program.elf -p 3000

3. 批处理

以事件驱动模式运行并输出 JSON 指标，适用于自动化管道：

./build/arm_cpu_sim -e -f elf program.elf -j -o output.json

4. 设计空间探索

以独立配置运行多个并行实例：

./build/arm_cpu_sim -e --multi-instance --num-instances 4 -f elf program.elf
务必报告文件路径

在报告仿真结果时，务必同时指定输入文件路径和输出文件路径，以确保结果完全可复现。
