常见问题与排错

ARM CPU 仿真器在构建、运行和可视化仿真过程中的常见问题及解决方案。

构建问题

Q: CMake 配置失败，提示 "CMake 3.20 or higher required"

A: 安装更新的 CMake：brew install cmake（macOS）或使用发行版的包管理器。

Q: 编译失败，出现 C++20 错误

A: 确保使用支持 C++20 的编译器：clang 14+、GCC 12+ 或 Apple Clang 15+。

Q: CMake 配置期间 Capstone 获取失败

A: Capstone v5 通过 CMake FetchContent 获取。确保首次构建时有网络连接。获取的依赖会缓存供后续构建使用。

Q: 找不到 liblzma（ChampSim XZ 支持）

A: 安装 XZ 工具：brew install xz（macOS）或 apt install liblzma-dev（Ubuntu）。仅 ChampSim XZ 格式需要此依赖。

仿真问题

Q: 仿真没有输出 / JSON 为空

A: 检查 Trace 文件是否存在且可读。使用 -v 获取详细输出。验证格式标志与文件类型匹配。

Q: ELF 仿真期间出现 "Unsupported instruction" 错误

A: Capstone 解码器支持约 500 种 ARM64 助记符。某些罕见或特权指令可能不受支持。检查报告的 PC 地址处的指令。

Q: 仿真运行时间过长 / 永不结束

A: 使用 -n（最大指令数）或 -c（最大周期数）限制仿真范围。使用 -e（Event 模式）进行更快的批量仿真，避免可视化开销。

Q: 每次运行结果不同

A: 对于相同的输入和配置，结果应该是确定性的。如果出现非确定性行为，请检查测试程序中是否存在未初始化的内存。

可视化问题

Q: 可视化页面空白 / 没有数据显示

A: 确保加载了正确的 output.json 文件。该文件必须由仿真运行生成（而非 Event 模式 -e，该模式不生成 Konata 数据）。检查浏览器控制台是否有 JavaScript 错误。

Q: Web 服务器模式 (-p) 无法工作

A: 确保端口未被占用。尝试其他端口：-p 8080。检查防火墙设置。

交叉编译问题

Q: 找不到 aarch64-elf-gcc

A: 通过 Homebrew 安装：brew install aarch64-elf-gcc（macOS）。在 Linux 上：apt install gcc-aarch64-linux-gnu。

Q: 动态链接的 ELF 加载失败

A: 动态 ELF 需要目标架构的 glibc sysroot。使用静态链接（-static -nostdlib）以确保可靠的仿真。

性能问题

Q: 仿真速度很慢

A: 尝试以下优化：

  • 使用 -e（Event 模式）跳过可视化开销
  • 使用 -n 减少最大指令数
  • 使用 --preset minimal 进行更快的测试
  • 使用 Release 构建：./scripts/build.sh Release

Q: 内存占用过高

A: 内存使用量与指令窗口大小和 Trace 长度成正比。减小 --window-size 或使用 -n 限制指令数。

仍然无法解决？

查看自动化脚本页面了解完整测试套件（./scripts/test_all.sh），它可以帮助判断问题是环境特定的还是仿真器的 Bug。
