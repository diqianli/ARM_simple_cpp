输入格式

仿真器支持四种输入格式，每种适用于不同的使用场景，从完整的 ARM64 二进制执行到轻量级 text trace。

格式对比

特性 ELF Text ChampSim JSON
ISA 覆盖 ~500 条助记符 ~40 条助记符 N/A（trace 格式） N/A（trace 格式）
需要 Capstone 是 否 否 否
分支预测 功能预仿真 无 Trace 编码 N/A
适用场景 完整仿真 快速调试 学术研究 自定义 Pipeline
CLI 参数 -f elf -f text -f champsim -f json

ELF 格式 推荐

ELF 格式提供最完整的仿真体验。它加载编译好的 ARM64 ELF 二进制文件，通过完整的 Pipeline 执行，并使用 Capstone v5 反汇编器解码指令。

主要特性

  • 完整的 ARM64 ISA 覆盖 — 约 500 条助记符，包括 ARMv8.0 到 ARMv8.7-A 扩展
  • SVE 和 SME 支持 — 可伸缩向量扩展（Scalable Vector Extension）和可伸缩矩阵扩展（Scalable Matrix Extension）指令
  • 加密扩展 — AES、SHA1、SHA2 和 CRC 指令
  • 功能预仿真 — FunctionalSim 组件在时序仿真之前运行快速功能仿真以解析分支目标，提供准确的分支预测
  • 符号解析 — 解析 ELF 符号表以识别函数边界
  • 静态和动态链接 — 同时支持静态链接和动态链接的二进制文件

用法

# 运行 ELF 二进制文件
./build/arm_cpu_sim -f elf program.elf

# 自定义输出和内存大小
./build/arm_cpu_sim -f elf program.elf -o result.json -m 512
工作原理

ELF 加载器解析二进制文件头，将代码段和数据段加载到模拟内存中，然后 Capstone v5 反汇编每条指令。在时序仿真开始前，功能预仿真阶段会解析所有分支目标，确保 Pipeline 行为准确。

Text 格式

Text 格式是一种简单、人类可读的指令 trace 格式。适用于快速调试、最小化测试以及不希望依赖 Capstone 或交叉编译器的场景。

ISA 覆盖

Text 解析器支持约 40 条助记符，涵盖最常见的指令类别：

  • 整数 ALU：ADD、SUB、MUL、AND、ORR、EOR、LSL、LSR、ASR、MOV、MVN、NEG
  • Load/Store：LDR、STR、LDP、STP、LDUR、STUR
  • 分支：B、BL、CBZ、CBNZ、B.EQ、B.NE、B.LT、B.GT、B.LE、B.GE、RET
  • 基本浮点：FADD、FSUB、FMUL、FDIV

行格式

OPCODE [dest,] src1[, src2]  [memory_addr]  [branch_target]

示例 Text Trace

用法

./build/arm_cpu_sim -f text trace.txt -o result.json
Text 模式无分支预测

Text 格式不包含分支目标信息，因此分支预测不可用。所有分支在时序仿真中按已跳转处理。如需准确的分支行为，请使用 ELF 格式。

ChampSim 格式

ChampSim 格式是计算机架构研究中广泛使用的学术 trace 格式，用于 Cache 和分支预测器评估。它将内存访问模式和分支结果编码为紧凑的二进制记录。

变体

  • champsim — 原始（未压缩）ChampSim trace 文件
  • champsim_xz — XZ 压缩的 ChampSim trace 文件（构建时需要 liblzma）

用法

# 原始 ChampSim trace
./build/arm_cpu_sim -f champsim trace.champsim

# XZ 压缩的 ChampSim trace
./build/arm_cpu_sim -f champsim_xz trace.champsim.xz
XZ 压缩依赖

champsim_xz 格式要求构建系统安装 liblzma 开发库。Ubuntu/Debian：apt install liblzma-dev。macOS：brew install xz。

JSON 格式

JSON 格式接受预格式化的 JSON 结构化指令 trace。适用于程序化生成 trace 或从其他仿真工具转换。

用法

./build/arm_cpu_sim -f json trace.json -o result.json

选择合适的格式

场景 推荐格式 原因
需要准确 Pipeline 行为的完整程序仿真 elf 完整的 ISA 覆盖，功能预仿真提供分支目标
快速调试或测试特定指令序列 text 无需交叉编译器或 Capstone 依赖，易于手写
Cache 层次结构或分支预测器研究 champsim 标准学术格式，trace 数据集广泛可用
从其他工具自定义生成 trace json 结构化格式，易于程序化生成
