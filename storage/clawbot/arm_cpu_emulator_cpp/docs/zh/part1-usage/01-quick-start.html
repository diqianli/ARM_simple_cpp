快速开始

在五分钟内完成 ARM CPU 仿真器的构建、编译并运行你的第一次仿真。

前置条件

构建和运行仿真器需要以下工具：

工具 最低版本 用途
CMake 3.20+ 构建系统配置
C++ 编译器 Clang 14+ / GCC 12+ C++20 语言支持
aarch64-elf-gcc 任意近期版本 交叉编译 ARM64 测试程序（使用 text trace 输入时为可选）
依赖库自动下载

所有库依赖（nlohmann/json、spdlog、GoogleTest、ankerl::unordered_dense、Capstone v5）均通过 CMake FetchContent 自动下载，无需手动安装。

构建仿真器

方式 A：一键构建（推荐）

项目提供了一个便捷脚本，自动完成 CMake 配置和编译，并自动检测并行度：

# Release 构建（默认）
./scripts/build.sh

# Debug 构建（含调试符号，无优化）
./scripts/build.sh Debug

方式 B：手动 CMake 构建

如果你更喜欢直接运行 CMake：

# 配置
cmake -B build -DCMAKE_BUILD_TYPE=Release

# 构建（使用所有可用核心）
cmake --build build -j$(nproc)
构建产物

编译后的二进制文件位于 build/arm_cpu_sim。测试二进制文件（test_config、test_ooo、test_simulation）同样在 build/ 目录下。

编译测试程序

仿真器执行 ARM64 ELF 二进制文件。使用提供的脚本编译内置测试源码：

./scripts/compile_test_elf.sh

该脚本会自动在 PATH 中搜索 aarch64-elf-gcc、aarch64-linux-gnu-gcc 或 aarch64-none-elf-gcc 交叉编译器。编译后的二进制文件位于 tests/data/test_elf_aarch64。

交叉编译器安装（macOS）

在 macOS 上，可通过 Homebrew 安装 aarch64-elf-gcc：brew install aarch64-elf-gcc

运行第一次仿真

使用编译好的测试 ELF 二进制文件执行仿真器：

# 输入文件：tests/data/test_elf_aarch64
# 输出文件：output.json（当前目录下）
./build/arm_cpu_sim -f elf tests/data/test_elf_aarch64

仿真完成后，仿真器会将结果写入当前工作目录下的 output.json。该文件包含仿真统计信息（IPC、CPI、Cache 命中率、指令混合、逐周期 Pipeline 追踪以及 Konata 可视化数据）。

务必记录文件路径

在报告仿真结果时，务必同时指定输入文件路径和输出文件路径，以确保结果可复现。

自定义输出

使用 -o 指定自定义输出路径：

./build/arm_cpu_sim -f elf tests/data/test_elf_aarch64 -o results/my_sim.json

使用 -n 限制仿真指令数量（适用于快速测试）：

./build/arm_cpu_sim -f elf tests/data/test_elf_aarch64 -n 1000

打开可视化

仿真器会生成兼容 Konata 的 Pipeline 时间线数据。可以通过两种方式查看：

方式 A：直接打开 HTML 文件

在浏览器中打开以下 URL：

file:///Users/mac/storage/clawbot/arm_cpu_emulator_cpp/tools/viz_server/index.html

然后通过可视化界面中的文件选择器加载输出的 JSON 文件。

方式 B：使用内置 Web 服务器

启动本地 HTTP 服务器，自动提供可视化页面：

./build/arm_cpu_sim -f elf tests/data/test_elf_aarch64 -p 3000

然后在浏览器中打开 http://localhost:3000。可视化页面会自动加载，输出数据也可在同一地址获取。

验证构建

运行单元测试以确认一切正常工作：

# 运行所有单元测试
cd build && ctest --output-on-failure

# 或运行完整测试套件
./scripts/test_all.sh
