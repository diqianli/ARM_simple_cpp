编写测试程序

如何为仿真器编写裸机 ARM64 汇编和 C 程序，涵盖指令模式、编译和执行环境细节。

裸机 ARM64 编程基础

仿真器在裸机环境中执行 ARM64 ELF 二进制文件。没有操作系统、没有 libc、没有系统调用接口。程序直接运行在模拟硬件上。

入口点

入口点必须是 _start 符号。ELF 加载器从该符号解析的地址开始执行。如果找不到 _start，加载器将使用 ELF 文件头中的入口点。

.global _start
_start:
    // 你的代码从这里开始
    MOV X0, #42
    // ...

ELF 段（Sections）

段 内容 说明
.text 可执行代码 加载到模拟内存中由 Pipeline 执行的指令
.data 已初始化数据 具有显式初始值的变量，从 ELF 文件加载
.bss 零初始化数据 初始值为零的变量，已分配但未存储在 ELF 文件中

寄存器约定（AAPCS64）

寄存器 用途 调用者/被调用者保存
X0 – X7 参数和返回值 调用者保存
X8 间接结果位置寄存器 调用者保存
X9 – X15 临时/暂存寄存器 调用者保存
X16 – X17 过程内调用暂存寄存器 调用者保存
X18 平台寄存器（保留） 因平台而异
X19 – X28 被调用者保存寄存器 被调用者保存
X29 帧指针（FP） 被调用者保存
X30 链接寄存器（LR） 调用者保存
SP 栈指针 特殊
XZR（WZR） 零寄存器（读始终为 0） N/A
程序终止

没有可用的 exit() 系统调用。仿真器在达到最大指令数（-n）或最大周期数（-c）时终止。请确保你的测试程序在这些限制内运行。

常用指令模式

算术指令

ADD X0, X1, X2      // X0 = X1 + X2
ADD X0, X1, #16     // X0 = X1 + 16（立即数操作数）
SUB X3, X4, #16     // X3 = X4 - 16（立即数操作数）
SUB X3, X4, X5      // X3 = X4 - X5
MUL X5, X6, X7      // X5 = X6 * X7
AND X8, X9, X10     // X8 = X9 & X10（按位与）
ORR X8, X9, X10     // X8 = X9 | X10（按位或）
EOR X8, X9, X10     // X8 = X9 ^ X10（按位异或）
LSL X8, X9, #3      // X8 = X9 << 3（逻辑左移）
LSR X8, X9, #5      // X8 = X9 >> 5（逻辑右移）
ASR X8, X9, #2      // X8 = X9 >> 2（算术右移）
MOV X0, X1           // X0 = X1（寄存器传送）
MOV X0, #42          // X0 = 42（立即数传送）
MVN X0, X1           // X0 = ~X1（按位取反）

内存访问指令

// Load 指令
LDR X0, [X1, #0]    // 从地址 X1+0 加载 64 位值到 X0
LDR X0, [X1, #8]    // 从地址 X1+8 加载 64 位值到 X0
LDR W0, [X1, #4]    // 从地址 X1+4 加载 32 位值到 W0
LDR X0, [X1, X2]    // 从地址 X1+X2 加载 64 位值（寄存器偏移）
LDUR X0, [X1, #-1]  // 非缩放偏移加载（支持负偏移）

// Store 指令
STR X2, [X3, #8]    // 将 64 位 X2 存储到地址 X3+8
STR W2, [X3, #4]    // 将 32 位 W2 存储到地址 X3+4
STUR X0, [X1, #-1]  // 非缩放偏移存储

// Load/Store Pair（一条指令操作两个寄存器）
LDP X0, X1, [X2]    // 从 [X2] 加载 X0，从 [X2+8] 加载 X1
LDP X0, X1, [X2, #16] // 从 [X2+16] 加载 X0，从 [X2+24] 加载 X1
STP X0, X1, [X2]    // 将 X0 存到 [X2]，将 X1 存到 [X2+8]
STP X0, X1, [X2, #16] // 将 X0 存到 [X2+16]，将 X1 存到 [X2+24]

// 前索引和后索引
LDR X0, [X1, #8]!   // 从 X1+8 加载，然后 X1 = X1+8（前索引）
LDR X0, [X1], #8    // 从 X1 加载，然后 X1 = X1+8（后索引）

分支指令

B label              // 无条件分支到 label
BL func              // 带链接的分支（将返回地址保存到 X30）
RET                  // 返回到 X30 中的地址

// 比较并分支
CBZ X0, label        // 如果 X0 == 0 则分支到 label
CBNZ X0, label       // 如果 X0 != 0 则分支到 label

// 条件分支（在 CMP 或 SUBS 之后）
B.EQ label           // 相等时分支（Z 标志置位）
B.NE label           // 不等时分支（Z 标志清零）
B.LT label           // 小于时分支（有符号）
B.GT label           // 大于时分支（有符号）
B.LE label           // 小于等于时分支（有符号）
B.GE label           // 大于等于时分支（有符号）
B.HI label           // 无符号高于时分支
B.LS label           // 无符号低于或等于时分支

// 比较指令（设置标志但不写回结果）
CMP X0, X1           // 根据 X0 - X1 设置标志
CMP X0, #42          // 根据 X0 - 42 设置标志

// 条件选择
CSEL X0, X1, X2, EQ  // X0 = (EQ) ? X1 : X2

浮点指令

FADD D0, D1, D2      // 双精度：D0 = D1 + D2
FSUB D0, D1, D2      // 双精度：D0 = D1 - D2
FMUL D0, D1, D2      // 双精度：D0 = D1 * D2
FDIV D0, D1, D2      // 双精度：D0 = D1 / D2
FADD S0, S1, S2      // 单精度：S0 = S1 + S2
FMUL S0, S1, S2      // 单精度：S0 = S1 * S2
FMOV D0, X0          // 整数到浮点寄存器的传送（重解释位模式）
FMOV X0, D0          // 浮点寄存器到整数寄存器的传送

SIMD 指令

// 128 位向量操作（NEON）
ADD V0.4S, V1.4S, V2.4S  // 4 x 32 位整数加法
ADD V0.2D, V1.2D, V2.2D  // 2 x 64 位整数加法
SUB V0.8B, V1.8B, V2.8B  // 8 x 8 位整数减法
MUL V0.4S, V1.4S, V2.4S  // 4 x 32 位整数乘法

// SIMD Load/Store
LDR Q0, [X1]         // 从内存加载 128 位向量
STR Q0, [X2]         // 将 128 位向量存储到内存

内存屏障指令

DMB ISH              // 数据内存屏障（Inner Shareable）
                       // 确保内存访问按顺序被
                       // Inner Shareable 域中的其他观察者感知

DSB ISH              // 数据同步屏障（Inner Shareable）
                       // 阻塞执行直到所有内存访问完成

ISB                  // 指令同步屏障
                       // 刷新指令 Pipeline，确保
                       // 后续指令在屏障之后取指

完整示例程序

以下示例演示了一个简单的循环，计算内存中的数组求和：

.global _start

.section .data
array:
    .quad 10, 20, 30, 40, 50    // 5 个元素
size = (. - array) / 8           // 元素数量

.section .bss
result:
    .skip 8                      // 64 位结果变量

.section .text
_start:
    LDR X0, =array               // X0 = 数组地址
    LDR X1, =size                // X1 = 元素数量
    LDR X2, =result              // X2 = 结果地址
    MOV X3, #0                   // X3 = 求和累加器 = 0
    MOV X4, #0                   // X4 = 循环索引 = 0

loop:
    CMP X4, X1                   // 比较索引与大小
    B.GE done                    // 如果索引 >= 大小，退出循环

    LDR X5, [X0, X4, LSL #3]    // 加载 array[index]（X4 * 8 字节偏移）
    ADD X3, X3, X5               // sum += array[index]
    ADD X4, X4, #1               // index++
    B loop                       // 重复

done:
    STR X3, [X2]                 // 将结果存储到内存
    B done                       // 无限循环（仿真器通过 -n 或 -c 终止）

编译

使用提供的脚本（推荐）

项目包含一个便捷脚本，自动检测交叉编译器并编译内置测试源码：

./scripts/compile_test_elf.sh

该脚本会在 PATH 中搜索 aarch64-elf-gcc、aarch64-linux-gnu-gcc 或 aarch64-none-elf-gcc，并将 scripts/test_elf_source.c 编译为 tests/data/test_elf_aarch64。

手动编译

# macOS（使用 Homebrew 的 aarch64-elf-gcc）
aarch64-elf-gcc -static -O2 -nostdlib -o program program.s

# Linux（使用 aarch64-linux-gnu-gcc）
aarch64-linux-gnu-gcc -static -O2 -nostdlib -o program program.s

# macOS/Linux（使用 aarch64-none-elf-gcc）
aarch64-none-elf-gcc -static -O2 -nostdlib -o program program.s

# 直接编译 C 源码（裸机）
aarch64-elf-gcc -static -O2 -nostdlib -o program program.c
编译器标志说明
  • -static — 静态链接（无动态库依赖）
  • -O2 — 优化级别 2（产生典型的真实世界指令模式）
  • -nostdlib — 不链接标准库（无 libc、无启动代码）

测试用优化级别

标志 效果 适用场景
-O0 无优化，指令字面映射 验证单条指令行为
-O1 基本优化 带有一些编译器变换的轻量测试
-O2 标准优化 推荐用于真实的仿真负载
-O3 激进优化，包括向量化 最大 ILP 和 SIMD 使用测试

内置测试源码

项目在 scripts/test_elf_source.c 中包含一个全面的测试源文件，覆盖所有主要指令类别：

  • ALU 操作 — ADD、SUB、MUL、AND、ORR、EOR、移位、传送
  • Load/Store — LDR、STR、LDP、STP，含多种寻址模式
  • 分支 — 条件分支、比较并分支、函数调用
  • 浮点运算 — FADD、FSUB、FMUL、FDIV，单精度和双精度
  • SIMD — NEON 指令的向量运算
  • 屏障 — DMB、DSB、ISB 内存和指令同步屏障

该文件由 scripts/compile_test_elf.sh 编译，生成的二进制文件位于 tests/data/test_elf_aarch64。

注意事项

模拟内存空间

程序运行在模拟的物理地址空间中。默认大小为 256 MB，可通过 -m <MB> 配置。所有代码、数据和栈必须在此空间内。如果程序访问超出分配范围的内存，仿真器将报告内存错误。

无 OS 服务

这是纯粹的裸机执行。没有系统调用、没有 printf、没有文件 I/O、没有定时器、没有中断处理。所有数据必须静态初始化或在程序运行时计算。

栈指针设置

栈指针（SP）由 ELF 加载器初始化到模拟内存区域的顶部。但如果你的程序使用深度嵌套的函数调用或大型栈帧，请确保栈有足够的空间。你也可以在 _start 例程中手动将 SP 设置为特定地址。

终止策略

由于没有退出机制，请在测试程序末尾设计一个无限循环（B .），并使用 -n <N> 或 -c <N> 限制仿真长度。或者，使用自然耗尽指令计数的有限循环。

运行你的程序

# 编译
aarch64-elf-gcc -static -O2 -nostdlib -o my_program my_program.s

# 仿真（输入：my_program，输出：output.json）
./build/arm_cpu_sim -f elf my_program -o output.json

# 仿真并限制指令数，同时启动可视化
./build/arm_cpu_sim -f elf my_program -n 10000 -p 3000 -o output.json
