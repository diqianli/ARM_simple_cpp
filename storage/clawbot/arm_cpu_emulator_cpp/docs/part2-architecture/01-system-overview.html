System Overview

High-level architecture of the ARM CPU Emulator: an ESL timing simulator for ARMv8 out-of-order processors.

The ARM CPU Emulator is an ESL (Electronic System Level) timing simulator for ARMv8 out-of-order processors. Written in C++20, ported from an original Rust implementation, it simulates instruction pipelines, cache hierarchies, and CHI interconnect protocols.

Module Architecture

The system is composed of several interconnected subsystems:

Subsystem Header Role
OoO Engine ooo/ooo_engine.hpp Instruction window, dependency tracking, reorder buffer
Memory memory/memory_subsystem.hpp L1/L2/L3 caches, LSQ, DDR controller
CHI chi/chi_manager.hpp AMBA CHI protocol (RNF, HNF, SNF nodes)
Decoder decoder/capstone_decoder.hpp Capstone v5 ARM64 disassembly (~500 mnemonics)
Input input/instruction_source.hpp Factory dispatching to format-specific parsers
Stats stats/stats_collector.hpp IPC, CPI, cache hit rates, MPKI, instruction mix
Visualization visualization/visualization_state.hpp Konata JSON export for pipeline timeline viewer
Analysis analysis/topdown.hpp TopDown microarchitectural analysis

Two Simulation Paths

The emulator provides two simulation modes that share common subsystems:

  1. CPUEmulator (cpu.hpp): Full path with CHI interconnect, Konata visualization export, and trace output. This is the default mode.
  2. SimulationEngine (simulation/simulation_engine.hpp): Event-based path emitting SimulationEvent to pluggable sinks. No Konata export. Enabled with -e flag.

Both paths share the same OoO engine, memory subsystem, and stats collector.

Data Flow

Instructions flow through the system as follows:

InstructionSource (trace/ELF input)
  → Fetch/Dispatch
  → OoOEngine (rename, issue, execute)
  → MemorySubsystem (LSQ, L1/L2/L3, DDR)
  → Commit
  → Stats/Visualization

Source Directory Structure

include/arm_cpu/
├─ cpu.hpp              # CPUEmulator main class
├─ config.hpp           # CPUConfig and presets
├─ types.hpp            # Core types (Instruction, Reg, OpcodeType)
├─ error.hpp            # Result<T> error handling
├─ decoder/             # Instruction decoders
├─ ooo/                 # Out-of-order engine
├─ memory/              # Cache hierarchy and DDR
├─ chi/                 # CHI coherence protocol
├─ input/               # Input source interfaces
├─ stats/               # Statistics collection
├─ visualization/       # Visualization and Konata export
└─ analysis/            # TopDown analysis
src/
├─ cpu.cpp              # CPUEmulator implementation
├─ main.cpp             # CLI entry point
├─ decoder/             # Decoder implementations
├─ ooo/                 # OoO engine implementations
├─ memory/              # Memory subsystem implementations
├─ input/               # Input source implementations
├─ stats/               # Stats implementations
└─ visualization/       # Visualization implementations
