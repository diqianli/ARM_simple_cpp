Visualization System

Generates Konata-compatible JSON for pipeline timeline visualization with interactive web-based rendering.

VisualizationState

Source file: visualization/visualization_state.hpp

The VisualizationState class is the central state manager for all visualization data. It maintains a list of VisualInstruction entries, where each entry contains:

  • Instruction metadata — PC, opcode, mnemonic, and operand string
  • Pipeline stage timestamps — cycle number when the instruction enters each stage (fetch, decode, rename, dispatch, issue, execute, memory, writeback, commit)
  • Color code — based on instruction type for visual differentiation in the timeline

The state manager also provides:

  • Snapshot management for incremental rendering of large traces
  • History recording for undo/redo navigation in the visualization interface

PipelineTracker

Source file: simulation/pipeline_tracker.hpp

The PipelineTracker records stage transitions during simulation. It uses a StageTiming struct that stores the cycle number when an instruction enters each pipeline stage. The tracker is called by the CPU pipeline at each stage transition:

Method Pipeline Stage
record_fetch()Instruction Fetch
record_decode()Decode
record_rename()Register Rename
record_dispatch()Dispatch / Allocate
record_issue()Issue / Schedule
record_execute()Execute
record_memory()Memory Access
record_writeback()Writeback
record_commit()Commit / Retire

Konata JSON Export

The visualization system generates JSON output in a format compatible with the Konata pipeline viewer. The structure organizes data by cycle, with each cycle containing the instructions present in each pipeline stage:

{
  "core": [
    {
      "cycle": 0,
      "fetch": [{"pc": 0x1000, "instr": "ADD X0, X1, X2"}],
      "decode": [...],
      "rename": [...],
      "dispatch": [...],
      "issue": [...],
      "execute": [...],
      "memory": [...],
      "writeback": [...],
      "commit": [...]
    }
  ]
}

Each instruction entry includes the program counter (PC) and the disassembled instruction string, allowing the viewer to display both address and mnemonic information for each pipeline slot.

Web Server

The visualization web interface is located in tools/viz_server/:

File Purpose
index.html Main visualization page with file picker and tabbed views
app_static.js JavaScript application logic for data loading and view management
style.css Visualization-specific styles (timeline, metrics panels, controls)
konata/konata_renderer.js Konata pipeline timeline renderer with zoom and pan support
perf_stats_view.js Tabular performance metrics view (IPC, cache stats, instruction mix)

The visualization can be accessed in two ways:

  • Built-in web server: Use -p <port> to start an HTTP server that serves the visualization page automatically at http://localhost:<port>
  • Static file: Open tools/viz_server/index.html directly in a browser and load the output JSON file through the file picker

Kanata Log Export

Source file: visualization/kanata_log_exporter.hpp

The KanataLogExporter provides an alternative export format for execution traces:

  • Exports execution trace in Kanata log format
  • Compatible with external Kanata visualization tools
  • Includes per-instruction stage timings and dependency information
  • Useful for cross-tool analysis and comparison with other simulator outputs
