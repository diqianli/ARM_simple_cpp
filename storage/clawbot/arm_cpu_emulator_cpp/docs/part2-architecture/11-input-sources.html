Input Sources

Pluggable input source architecture that supports multiple trace formats, with a factory function that dispatches to the appropriate parser.

The simulator uses a pluggable input source architecture. A factory function create_source() dispatches to the appropriate parser based on the specified format, allowing new input types to be added with minimal changes to the core simulation loop.

InstructionSource Interface

Defined in input/instruction_source.hpp, this abstract base class defines the contract for all input sources:

  • has_next(): Check if more instructions are available in the trace
  • next(): Fetch the next instruction from the trace
  • peek(): Look at the next instruction without consuming it
  • reset(): Reset to the beginning of the trace

Input Source Implementations

1. TraceInput (input/instruction_source.cpp)

Reads instructions from human-readable text trace files. Parses instructions line-by-line in the format OPCODE [dest,] src1[, src2].

  • Format flag: -f text
  • Coverage: ~40 mnemonics (integer ALU, load/store, branch, basic FP)
  • Use case: Quick debugging and manual trace creation

2. ElfTrace (input/elf_trace.hpp, input/elf_trace.cpp)

Executes ELF binaries by disassembling code sections at runtime. This is the recommended input source for full ISA coverage.

  • Format flag: -f elf
  • Decoder: Uses Capstone decoder for instruction decode (~500 ARM64 mnemonics)
  • Branch prediction: Implements functional pre-simulation via FunctionalSim for accurate branch target resolution
  • Loop replay: Supports loop replay for long-running benchmarks

3. FunctionalSim (input/functional_sim.hpp, input/functional_sim.cpp)

Pre-executes the ELF binary functionally (without timing simulation) to determine actual branch outcomes. This provides "perfect branch prediction" data to the ElfTrace source.

  • Role: Records actual branch outcomes before the timing simulation runs
  • Output: Generates a trace of instructions with correct branch targets
  • Used by: ElfTrace internally for branch prediction

4. ChampSimTrace (input/champsim_trace.cpp)

Parses ChampSim academic trace format, commonly used in computer architecture research.

  • Format flag: -f champsim or -f champsim_xz
  • Compression: Supports both raw and XZ-compressed traces (via liblzma)
  • Format: 0 addr PCS accesses per line

Factory Function

The factory function creates the appropriate input source based on the configuration:

std::unique_ptr<InstructionSource> create_source(
    const TraceInputConfig& config,
    const CPUConfig& cpu_config);

It dispatches based on config.format (a TraceFormat enum), selecting the correct parser and decoder combination for the specified input type.

ELF Loop Replay

For benchmarks that run in tight loops, the simulator can replay the ELF multiple times to prevent the trace from ending prematurely. This is configured via max instructions (-n) or max cycles (-c) limits, which control when the simulation terminates regardless of the trace length.
