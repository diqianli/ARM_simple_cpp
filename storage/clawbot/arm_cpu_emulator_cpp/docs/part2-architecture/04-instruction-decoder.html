Instruction Decoder

Translates ARM64 instruction encodings into internal structures for pipeline execution.

The decoder subsystem translates instruction encodings into internal Instruction structures. Two decoder paths are available, selected automatically based on input format.

Capstone Decoder

The primary decoder (decoder/capstone_decoder.hpp) wraps Capstone v5 for full ARM64 disassembly:

  • Wraps Capstone v5 disassembly engine for ARM64
  • Covers ARMv8.0 through ARMv8.7-A
  • Supports extensions: SVE, SME, Crypto (AES, SHA1, SHA2, SHA3)
  • ~500 mnemonic mappings via MnemonicLookup table
  • Detects conditional branches for branch prediction

Key flow:

  1. Capstone disassembles instruction bytes
  2. Mnemonic string mapped to internal OpcodeType enum
  3. Operands parsed (registers, immediates, memory addresses)
  4. Instruction built using builder pattern

Custom AArch64 Decoder

A lightweight fallback decoder (decoder/aarch64_decoder.hpp) for when Capstone is not available:

  • Sub-decoders: arithmetic, logical, load_store, branch, simd_neon, fp, crypto, system
  • Used as fallback when Capstone is not available
  • DecodedInstruction struct with builder pattern

Instruction Builder Pattern

Instructions are constructed using a fluent builder pattern:

Instruction instr;
instr.with_opcode(OpcodeType::Add)
     .with_pc(pc)
     .with_src_reg(Reg::X1)
     .with_src_reg(Reg::X2)
     .with_dst_reg(Reg::X0);

Conditional Branch Detection

The Capstone decoder identifies conditional branches (B.cond, CBZ, CBNZ, TBZ, TBNZ) and sets branch prediction metadata for the pipeline. This information flows into the OoO engine to enable speculative execution of predicted-taken paths.
